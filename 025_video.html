🚀 Introducción

🎯 El Comando Más Importante

El comando SELECT es la herramienta fundamental para recuperar información de la base de datos. Aunque su sintaxis puede parecer compleja, nos permite una gran flexibilidad para decidir exactamente qué queremos recuperar.

💡 Flexibilidad del SELECT

No te dejes engañar por su aparente simplicidad. SELECT nos permite una gran flexibilidad para construir consultas desde las más básicas hasta las más complejas, abarcando el 95% de situaciones que encontrarás en el trabajo real.

📖 Notaciones de Sintaxis

Para interpretar correctamente la sintaxis:

  • [texto]: Entre corchetes = opcional
  • opción1 | opción2: Separado por | = elegir una opción
  • MAYÚSCULAS: Palabras clave del SQL
  • minúsculas: Nombres de tablas, campos, etc.

🔄 Metodología de Consultas

Para construir cualquier consulta, siempre debemos respondernos tres preguntas fundamentales que nos guiarán hacia la solución correcta. Esta metodología es especialmente importante para principiantes.

📌 Enfoque Práctico:

Vamos paso a paso, tratando de no complicar las cosas. Veremos los comandos más comunes con ejemplos prácticos que podrás ejecutar y modificar para experimentar.

📝 Sintaxis Completa del SELECT

🔧 Sintaxis Completa

Esta es la sintaxis completa del comando SELECT. No te preocupes si parece abrumadora; iremos viendo cada parte paso a paso:

[WITH subconsulta AS (SELECT ...)] SELECT [TOP número] [ALL | DISTINCT] columna1 [AS alias1], columna2 [AS alias2] FROM tabla [AS alias] [INNER | LEFT | RIGHT | FULL] JOIN tabla2 ON condición [WHERE condición] [GROUP BY expresión] [HAVING condición_agrupación] [ORDER BY campo [ASC | DESC]]

📋 Orden de Ejecución

Es importante comprender el orden lógico en que SQL procesa las cláusulas:

  1. FROM: Identifica las tablas fuente
  2. WHERE: Filtra registros individuales
  3. GROUP BY: Agrupa los datos
  4. HAVING: Filtra grupos
  5. SELECT: Selecciona las columnas
  6. ORDER BY: Ordena el resultado final

✅ Forma Más Básica

La estructura más simple del SELECT:

SELECT * FROM tabla;

🔍 Desglose de la Sintaxis Básica

  • SELECT: Indica que queremos recuperar información
  • * (asterisco): Comodín para "todos los campos"
  • FROM: Indica el origen de datos
  • tabla: Nombre del objeto de base de datos
⚠️ Importante:

El orden de las cláusulas es obligatorio. Si no respetas el orden, obtendrás errores de sintaxis. Siempre FROM antes que WHERE, GROUP BY después de WHERE, etc.

⭐ Uso Básico del SELECT

🔍 Exploración de Tablas

Cuando no conocemos una tabla, es útil hacer una consulta completa para ver su estructura y datos. Esto nos ayuda a entender qué información contiene y cómo está organizada.

-- Ver toda la información de la tabla empleados SELECT * FROM empleados;
📊 Filtrado de Columnas

En lugar del asterisco (*), podemos especificar exactamente qué columnas queremos ver. Esto es más eficiente y claro:

-- Mostrar solo campos específicos SELECT id_empleado, tratamiento, nombre, apellidos FROM empleados;

💡 Literales en SELECT

También podemos incluir texto literal en nuestras consultas para hacer los resultados más legibles:

-- Añadir etiquetas literales SELECT 'Identificador:', id_empleado, 'Nombre:', nombre, 'Apellidos:', apellidos FROM empleados;

🏷️ Alias con AS

Los alias nos permiten renombrar columnas o tablas para mayor claridad:

-- Usar alias para claridad SELECT id_empleado AS "Identificador", nombre AS "Nombre del Empleado", apellidos AS "Apellidos" FROM empleados;
💡 Consejo Práctico:

Usar SELECT * está bien para exploración inicial, pero cuando necesites resultados específicos, delimita siempre la salida especificando las columnas exactas que necesitas.

🎯 Cuándo Usar Cada Enfoque

  • SELECT *: Para explorar tablas desconocidas
  • Columnas específicas: Para consultas de producción
  • Literales: Para reportes con etiquetas
  • Alias: Para mayor claridad en los resultados

🔍 Filtrado de Datos

🎯 Cláusula WHERE

La cláusula WHERE es fundamental para filtrar registros. Nos permite especificar condiciones que deben cumplir los datos que queremos recuperar.

-- Filtrar empleados por tratamiento SELECT id_empleado, tratamiento, nombre, apellidos FROM empleados WHERE tratamiento = 'Srta.';
🔢 Limitación con TOP

La cláusula TOP limita el número de registros devueltos. Es útil para pruebas o cuando solo necesitas una muestra:

-- Mostrar solo los primeros 5 empleados SELECT TOP 5 * FROM empleados; -- Si hay menos registros, muestra todos los disponibles SELECT TOP 15 * FROM empleados; -- Solo hay 9, muestra los 9
🎭 ALL vs DISTINCT

Controlan cómo se manejan los valores duplicados en los resultados:

  • ALL: Opción por defecto, muestra todos los registros
  • DISTINCT: Elimina registros duplicados
-- Mostrar todos los clientes (con duplicados) SELECT ALL id_cliente FROM pedidos; -- 772 registros -- Mostrar solo clientes únicos (sin duplicados) SELECT DISTINCT id_cliente FROM pedidos; -- 89 registros únicos
🔍 Filtros Avanzados

WHERE puede usar operadores más complejos:

-- Clientes cuyo ID comience con 'A' SELECT * FROM pedidos WHERE id_cliente LIKE 'A%'; -- Pedidos con cargo mayor a 35 SELECT * FROM pedidos WHERE cargo > 35;
🔑 Diferencia Importante

TOP es limitación, no filtro: TOP simplemente toma los primeros N registros que encuentra, no calcula cuáles son los "mejores". Para filtros inteligentes usa WHERE.

⚠️ Atención con Literales:

Cuando uses literales de texto en WHERE, las mayúsculas y minúsculas sí importan. 'Srta.' es diferente de 'srta.' o 'SRTA.'

📊 Agrupación y Funciones

📈 Funciones de Agregación

Las funciones de agregación realizan operaciones matemáticas sobre conjuntos de datos, permitiendo obtener resúmenes estadísticos.

COUNT()

Conteo

Cuenta el número de registros

SUM()

Suma

Suma valores numéricos

AVG()

Promedio

Calcula la media aritmética

MAX()

Máximo

Encuentra el valor más alto

MIN()

Mínimo

Encuentra el valor más bajo

📊 Ejemplo Básico de Agregación

Operaciones sobre toda la tabla:

-- Estadísticas generales de pedidos SELECT COUNT(id_pedido) AS "Total Pedidos", SUM(cargo) AS "Total Cargo" FROM pedidos;
🏗️ GROUP BY - Agrupación

GROUP BY permite agrupar datos por criterios específicos y aplicar funciones de agregación a cada grupo:

-- Estadísticas por cliente SELECT id_cliente, COUNT(id_pedido) AS "Número de Pedidos", SUM(cargo) AS "Total Gastado" FROM pedidos GROUP BY id_cliente;
🔑 Regla Fundamental de GROUP BY

Todos los campos del SELECT que NO sean funciones de agregación deben aparecer en GROUP BY. Si tienes SELECT id_cliente, nombre, COUNT(*), entonces GROUP BY debe incluir id_cliente, nombre.

🎯 HAVING - Filtrado de Grupos

HAVING es como WHERE, pero filtra después de la agrupación. Permite filtrar grupos basándose en los resultados de las funciones de agregación:

-- Solo clientes con más de 10 pedidos SELECT id_cliente, COUNT(id_pedido) AS "Número de Pedidos" FROM pedidos GROUP BY id_cliente HAVING COUNT(id_pedido) > 10;
🔄 WHERE vs HAVING

Diferencias clave en el procesamiento:

  • WHERE: Filtra registros individuales antes de agrupar
  • HAVING: Filtra grupos después de agrupar y agregar
-- Combinando WHERE y HAVING SELECT id_cliente, COUNT(id_pedido) AS "Pedidos", SUM(cargo) AS "Total" FROM pedidos WHERE cargo > 35 -- Primero: solo pedidos > 35 GROUP BY id_cliente HAVING SUM(cargo) < 250; -- Después: solo totales < 250
⚠️ Orden Importante:

Las cláusulas deben ir en orden específico: FROM → WHERE → GROUP BY → HAVING → ORDER BY. Cambiar este orden resultará en errores de sintaxis.

🔄 Ordenación y Subconsultas

📋 ORDER BY - Ordenación

La cláusula ORDER BY nos permite ordenar los resultados según uno o varios campos, en orden ascendente o descendente.

-- Ordenación simple SELECT id_cliente, fecha_pedido FROM pedidos ORDER BY id_cliente; -- Por defecto: ascendente -- Ordenación descendente SELECT id_cliente, fecha_pedido FROM pedidos ORDER BY id_cliente DESC; -- Explícitamente descendente
🔄 Ordenación Múltiple

Puedes ordenar por varios campos a la vez. SQL ordena por el primer campo, luego por el segundo dentro de cada grupo del primero, y así sucesivamente:

-- Ordenación por múltiples campos SELECT id_cliente, fecha_pedido FROM pedidos ORDER BY id_cliente DESC, -- Primero por cliente (Z→A) fecha_pedido ASC; -- Luego por fecha (antigua→nueva)
🎯 Lógica de Ordenación Múltiple

Con múltiples campos, SQL:

  1. Ordena todos los registros por el primer campo
  2. Dentro de cada valor del primer campo, ordena por el segundo
  3. Dentro de cada combinación, ordena por el tercero, etc.
📝 WITH - Subconsultas Nombradas

La cláusula WITH permite crear subconsultas con nombre que pueden reutilizarse en la consulta principal:

-- Subconsulta con WITH WITH subconsulta AS ( SELECT * FROM productos WHERE precio > 20 ) SELECT nombre, precio FROM subconsulta WHERE categoria = 'Bebidas';
💡 Cuándo Usar WITH

WITH es útil cuando: Tienes consultas complejas que necesitas reutilizar múltiples veces, o cuando quieres hacer más legible una consulta dividiendo la lógica en pasos.

🔍 Exploración de Estructura

Además de SELECT *, puedes explorar la estructura de tablas haciendo doble clic en el explorador de base de datos para ver:

  • Definición: Tipos de datos, claves primarias, valores por defecto
  • Índices: Optimizaciones de rendimiento aplicadas
  • Datos: Muestra del contenido actual
🎨 Trucos de Formato

Para consultas complejas:

  • Divide las líneas para mayor legibilidad
  • Usa indentación consistente
  • Agrupa cláusulas relacionadas
  • Comenta secciones complejas
💡 Buenas Prácticas:

ORDER BY siempre debe ser la última cláusula en tu SELECT. Úsalo cuando necesites un orden específico, pero recuerda que ordenar grandes conjuntos de datos puede afectar el rendimiento.
