🎯 Objetivo del Módulo

En esta clase nos centraremos en el SAP HANA Studio, la interfaz original de HANA que era el interface principal en la época de HANA 1.0 y que sigue estando en uso aunque con limitaciones.

🚀 Introducción a SAP HANA Studio

🎯 ¿Qué es SAP HANA Studio?

SAP HANA Studio es la interfaz original de HANA y aún es utilizada hoy en día, especialmente para clientes que ejecutan SAP HANA 1.0.

🏗️ Características Principales

  • Interfaz original - Primera herramienta de desarrollo para HANA
  • Basada en Eclipse - Plataforma de desarrollo conocida
  • Aplicación de escritorio - Se instala localmente
  • Motor XS clásico - Para desarrollo en HANA 1.0
  • Totalmente funcional - Administración, modelado y desarrollo

🔧 Funcionalidades Integradas

El SAP HANA Studio está integrado en el sistema de SAP HANA y provee un entorno completo para:

  • Administración - Gestión del sistema HANA
  • Modelado - Creación de vistas de información
  • Desarrollo - Aplicaciones y procedimientos
  • Aprovisionamiento de datos - Importación y gestión
📌 Limitaciones Actuales

SAP HANA Studio puede utilizarse en HANA 2.0 cuando se trabaja en modo desarrollo del motor XS clásico, pero no puede utilizarse para desarrollar aplicaciones en el motor XSA.

⚠️ Incompatibilidad de Motores

Debido a los cambios entre motores, no podremos ver desde un entorno (Studio) lo que creemos con el otro entorno (Web IDE). Se marca una diferencia clara y debemos tener muy en cuenta con qué motor queremos trabajar.

📈 Historia y Evolución de HANA

🌟 El Nacimiento de HANA

Cuando HANA apareció en el mercado hace relativamente poco, echó por tierra todo lo que se sabía y se sacó de la manga esta base de datos con nuevas tecnologías que no se habían visto hasta ahora:

  • Almacenaje in-memory - Datos en memoria RAM
  • Procesado en paralelo - Aprovechamiento de múltiples cores
  • Servidor web integrado - Motor XS incluido
  • Totalmente independiente - Ecosistema completo

🔄 Evolución y Mejoras

Como cualquier tecnología recién nacida, fue evolucionando con el tiempo:

🏷️ Nomenclatura y Versiones

HANA 1.0

  • Versiones: Hasta SPS 12
  • Motor: XS clásico
  • Interfaz: HANA Studio principal
  • Estado: Legacy

HANA 2.0

  • Versiones: Desde SPS 13+
  • Motores: XS + XSA
  • Interfaz: Web IDE principal
  • Estado: Actual
🔄 Convivencia de Motores

HANA 2.0 posee ambos motores (XS y XSA). La idea es hacerlos convivir un tiempo y más adelante cambiar totalmente al motor XSA.

🎯 Impacto en las Interfaces

  • HANA Studio → Conecta con motor XS
  • Web IDE → Conecta con motor XSA
  • Incompatibilidad → No se pueden ver objetos entre motores
  • Decisión empresarial → La empresa dictará qué motor usar

🔧 Base Tecnológica: Eclipse

☕ ¿Qué es Eclipse?

SAP HANA Studio es una aplicación basada en Java que corre sobre la plataforma Eclipse. Eclipse es una plataforma de programación de código abierto que se puede utilizar prácticamente para todo.

🌟 Características de Eclipse

  • Origen Java - Creado inicialmente para programación Java
  • Multiplataforma - Funciona en Windows, Linux, macOS
  • Extensible - Sistema de plugins muy potente
  • Código abierto - Libre y gratuito
  • Ampliamente usado - Por muchas empresas y proyectos

🔌 Sistema de Plugins

Al ser de código abierto, las empresas se ocupan de crear plugins que podemos instalar en Eclipse:

Ejemplos de Uso de Eclipse

  • Android - En sus inicios, el entorno de desarrollo era Eclipse
  • SAP ABAP - Se puede programar desde Eclipse en lugar del SAP GUI
  • .NET - Incluso se ha configurado para .NET (aunque no muy común)
  • SAP HANA - Plugins oficiales de SAP para desarrollo

⚙️ Instalación de HANA Studio

Para usar SAP HANA Studio necesitas:

  1. Instalar Eclipse (plataforma base)
  2. Descargar plugins SAP desde la página oficial
  3. Configurar plugins en Eclipse
  4. Crear conexión al sistema HANA
  5. Configurar credenciales de acceso

🔗 Conexión al Sistema

Cuando inicias SAP HANA Studio por primera vez, debes crear una conexión al sistema SAP HANA con el que vas a trabajar. Para ello necesitas:

  • Host y instancia - Dirección del servidor
  • Usuario y password - Credenciales válidas
  • Privilegios suficientes - Para acceder a lo que necesites
🟢 Indicador de Conexión

Una vez configurada la conexión, aparecerá una línea en el navegador. Si el botoncito está en verde, indica que estamos conectados a la base de datos y que esta se encuentra levantada y operativa.

  • Resetear - Windows → Perspective → Reset Perspective
  • Personalizar barras - Windows → Perspective → Customize
💡 Consejo Práctico

Es común que a base de mover elementos se desorganice todo. Siempre puedes usar Reset Perspective para volver a la configuración original si algo sale mal.

🎨 Perspectiva de Modelado

🎯 Perspectiva Más Importante

La perspectiva de modelado es la que más nos interesa. Está configurada especialmente para las tareas de arquitectura de datos y se recomienda cuando queremos crear modelos de información.

📊 Modelo Relacional vs Esquema en Estrella

Aquí se introduce un concepto importante sobre diferentes enfoques de estructuración de datos:

🔗 Modelo Relacional

  • Sistemas transaccionales
  • Reduce redundancia
  • Optimiza espacio
  • CRUD eficiente
  • Consultas específicas

⭐ Esquema en Estrella

  • Análisis de datos
  • Tabla central numérica
  • Tablas contextuales
  • Consultas analíticas
  • Velocidad superior

📈 Ejemplos de Análisis

El esquema en estrella se usa para casos como:

  • Gráficas de contagios COVID - Evolución temporal
  • Trackers de póker - Estadísticas de jugadores
  • Análisis empresarial - Patrones de compra
  • Business Intelligence - Dashboards ejecutivos

🔄 Evolución: BW vs HANA

📊 Business Warehouse (Tradicional)

  • Estructura física en BD
  • Dos servidores separados
  • Trasbase de datos
  • Tiempo y recursos

⚡ HANA (Moderno)

  • Estructura lógica
  • Un solo sistema
  • Tiempo real
  • Eficiencia superior
🎯 Enfoque del Curso

Estos conceptos de modelado analítico se verán más en detalle más adelante en el curso. Por ahora es importante entender que la perspectiva de modelado nos da las herramientas para crear estos tipos de estructuras.

🛠️ Herramientas Incluidas

La perspectiva de modelado incluye:

  • Consola SQL - Para escribir consultas
  • Árbol de estructuras - Navegador de BD
  • Herramientas de modelado - Para vistas de información
  • Gestión de objetos - Tablas, vistas, procedimientos, etc.

💾 Conceptos Fundamentales de Almacenamiento

🚨 PREGUNTA DE EXAMEN

Esta información es fundamental y puede aparecer en exámenes de certificación SAP. Si alguien falla esta pregunta, ¡tendrá consecuencias!

🎯 Concepto Clave: Almacenamiento Físico

El único objeto de la base de datos que almacena información (datos) de forma física son las TABLAS.

📊 ¿Qué Hace el Resto de Objetos?

Todo el resto de objetos nos muestran esa información, pero no la almacenan:

  • Vistas - Accedemos y vemos datos, pero no los almacenan
  • Procedimientos - Hacen operaciones y muestran datos
  • Funciones - Procesan y retornan datos
  • Todos acuden a las tablas para buscar los datos físicos
🔑 Principio Fundamental

Todos los objetos (excepto tablas) acuden a las tablas para buscar los datos, luego trabajan con ellos y nos los muestran. Pero SOLO LAS TABLAS ALMACENAN DATOS FÍSICAMENTE.

🏗️ Implicaciones en HANA

  • Modelos de información dan como resultado vistas de BD
  • SAP HANA no persiste datos redundantes para cada modelo
  • No crea agregados materializados físicos
  • Eficiencia en almacenamiento y gestión

💡 Ventajas de este Enfoque

  • Sin duplicación - Un solo lugar para cada dato
  • Consistencia - Cambios se reflejan en todas las vistas
  • Eficiencia de espacio - No redundancia de almacenamiento
  • Mantenimiento simplificado - Actualizaciones centralizadas
📝 Para Examen de Certificación

Memoriza: "Las tablas físicas son el único área de almacenamiento de datos dentro de SAP HANA. Todos los modelos de información resultan en vistas que no almacenan datos físicamente."

🌐 Gestión de Sistemas

➕ Agregar Sistema a HANA Studio

Para integrar un sistema SAP HANA en el Studio, necesitas conocer:

  • Servidor - Donde el motor está funcionando
  • Número de instancia - Puerto específico
  • Usuario y password - Credenciales válidas
  • Privilegios suficientes - Para establecer conexión

📋 Proceso de Configuración

  1. Acceder al asistente - Icono de servidores con "+"
  2. Introducir datos - Host, instancia, modo conexión
  3. Configurar credenciales - Usuario y contraseña
  4. Seleccionar esquema - Donde guardar tablas creadas
  5. Probar conexión - Verificar acceso

🗂️ Organización con Esquemas

Durante la importación de metadatos:

  • Se define qué esquema guardará las tablas creadas
  • Diferentes esquemas pueden agrupar tablas en categorías
  • Simplifica identificación - Qué tablas utilizar
  • No limita capacidades - Un modelo puede usar varios esquemas

📤 Import/Export de Landscapes

Puedes gestionar configuraciones de sistemas:

  • Importar landscape - File → Import → SAP HANA Studio Landscape
  • Exportar configuración - Para reutilizar en otros Studios
  • Gestionar passwords - Clic derecho → Database Logon User
  • Múltiples sistemas - Un Studio puede conectar a varios

🔄 Gestión de Múltiples Sistemas

HANA Studio puede manejar múltiples conexiones:

  • Varios sistemas en un solo Studio
  • Cada sistema con su propio árbol de objetos
  • Cambio fácil entre sistemas
  • Configuraciones independientes por sistema
💡 Consejo Práctico

Mantén organizadas tus conexiones con nombres descriptivos y documenta las credenciales de acceso para cada sistema (desarrollo, calidad, producción).

📦 Unidades de Entrega

📦 ¿Qué son las Unidades de Entrega?

Las unidades de entrega (Delivery Units) nos permiten ir almacenando los cambios que vayamos realizando en el servidor HANA para posteriormente exportar estas modificaciones a otros sistemas.

🔄 Equivalencia con SAP

Es el equivalente a las órdenes de transporte en SAP tradicional:

  • Desarrollo → Calidad → Producción
  • Fichero contenedor - Con todos los cambios
  • Despliegue conjunto - Todos los objetos relacionados
  • Control de versiones - Gestión de cambios

🛠️ Creación de Unidades de Entrega

Se crean desde la perspectiva SAP HANA Modeller:

  1. Abrir Quick View - Help → Quick View
  2. Delivery Units - Seleccionar sistema
  3. Crear nueva unidad - Siguiente
  4. Metadatos - Nombre, descripción, responsable
  5. Asignar objetos - Al crear nuevos elementos

📋 Metadatos de las Unidades

Al crear una unidad de entrega especificas:

  • Nombre - Identificador único
  • Descripción - Qué contiene la entrega
  • Unidad de envío - Asociación organizativa
  • Lenguaje original - Del contenido
  • Usuario responsable - Owner de los cambios
⚠️ Evolución en HANA 2.0

Esto funcionaba así en HANA 1.0. Cuando trabajamos con Web IDE en HANA 2.0, esto ha desaparecido y utilizamos los contenedores HDI que ofrecen funcionalidad similar pero con tecnología más moderna.

🔄 Flujo de Trabajo

  1. Crear unidad de entrega al inicio del proyecto
  2. Asignar objetos a la unidad al crearlos
  3. Desarrollar y probar en entorno de desarrollo
  4. Exportar unidad cuando esté lista
  5. Importar en destino (calidad/producción)
  6. Verificar despliegue correcto

💡 Ventajas del Sistema

  • Gestión integrada - Todo en una unidad
  • Dependencias controladas - Objetos relacionados juntos
  • Rollback posible - En caso de problemas
  • Trazabilidad - Qué cambios contiene cada entrega

🌐 Web Development Workbench

⚠️ Herramienta Legacy

Además del SAP HANA Studio, para desarrollos en XS existía una herramienta de desarrollo basada en web. Esto es solo para desarrollo y solo para motor XS.

🎯 Características del Web Workbench

  • Interfaz liviana - Basada en navegador web
  • Solo para desarrolladores - No para administración
  • Alternativa al Studio - Más ligera que HANA Studio
  • Funciones específicas - Solo lo que necesitan desarrolladores
  • Motor XS únicamente - No compatible con XSA

🔗 Acceso al Web Workbench

Para acceder necesitas abrir un navegador e introducir la URL:

https://<hostname>:<puerto>80<instancia>/hana/xs/ide/

Donde sustituyes:

  • <hostname> - Nombre del servidor
  • <instancia> - Número de instancia
🚨 Importante: No Confundir

Hay que tener muchísimo cuidado de no confundirlo con el Web IDE:

  • Web Development Workbench - Motor XS (HANA 1.0)
  • Web IDE - Motor XSA (HANA 2.0)
  • Son diferentes - Diferentes motores, diferentes capacidades
  • Pregunta trampa - Puede aparecer en exámenes

🔄 Proceso de Acceso

  1. Abrir navegador web
  2. Introducir URL correcta con parámetros del sistema
  3. Introducir credenciales - Usuario y contraseña
  4. Clic en Login
  5. Acceso al entorno de desarrollo web
📚 Para Exámenes de Certificación

Aunque no se han visto preguntas relacionadas desde hace mucho tiempo, SAP puede incluir preguntas trampa diferenciando entre Web Development Workbench (XS) y Web IDE (XSA).

📉 Estado Actual

  • Prácticamente desaparecido - De documentación y exámenes
  • Solo para sistemas legacy - HANA 1.0 antiguos
  • Reemplazado por Web IDE - Para nuevos desarrollos
  • Conocimiento histórico - Importante para contexto
