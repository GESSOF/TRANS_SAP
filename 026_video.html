🎯 Objetivos del Módulo

En esta clase introducimos el comando INSERT de SQL, que nos permite agregar nuevos registros a las tablas de nuestra base de datos. Aprenderemos las diferentes formas de insertar datos y los errores más comunes.

🔑 Concepto Fundamental

INSERT: Comando del lenguaje DML que sirve para introducir nuevos registros en las tablas de la base de datos. Su sintaxis es mucho más sencilla que SELECT.

🚀 Introducción

🎯 Propósito del INSERT

El comando INSERT es fundamental para alimentar nuestras tablas con datos. Como su nombre indica, nos sirve para insertar nuevos registros en las tablas de nuestra base de datos.

💡 Simplicidad vs SELECT

A diferencia del comando SELECT que tiene una sintaxis muy compleja, el INSERT es mucho más sencillo. Su estructura es directa y fácil de comprender, pero requiere precisión en los tipos de datos.

🛠️ Preparación Práctica

Para los ejemplos prácticos, crearemos una tabla de prueba que nos permitirá experimentar con diferentes formas de inserción de datos:

-- Crear tabla de prueba (adelantando DDL) CREATE COLUMN TABLE prueba_insert ( campo1 INTEGER, campo2 VARCHAR(50) );
🔍 Explorando la Estructura

Antes de insertar datos, es importante conocer la estructura de la tabla:

  • Doble clic en la tabla: Ver definición completa
  • SELECT * FROM tabla: Ver orden de campos
  • Tipos de datos: INTEGER, VARCHAR, etc.
📌 Nota Importante:

En este módulo nos adelantamos ligeramente al usar CREATE TABLE, que pertenece al lenguaje DDL. Lo veremos en detalle en la siguiente unidad.

📝 Sintaxis del INSERT

🔧 Sintaxis Completa

La sintaxis del comando INSERT es directa y comprensible:

INSERT INTO nombre_tabla [(columna1, columna2, ...)] VALUES (valor1, valor2, ...);
📋 Componentes de la Sintaxis
  • INSERT INTO: Palabra clave que indica la operación
  • nombre_tabla: Tabla donde insertar los datos
  • (columnas): Lista opcional de campos específicos
  • VALUES: Palabra clave que precede los valores
  • (valores): Datos a insertar en el orden correspondiente
✅ Ejemplo Básico

Forma más simple de insertar datos:

-- Inserción básica (todos los campos en orden) INSERT INTO prueba_insert VALUES (1, 'valor uno');
🎯 Dos Enfoques de Inserción
  1. Sin especificar columnas: Requiere valores para todos los campos en el orden exacto de la tabla
  2. Especificando columnas: Permite insertar valores desordenados o solo en algunos campos
🔑 Reglas Importantes

Si no especificas columnas: Debes proporcionar valores para todos los campos en el orden exacto que tienen en la tabla. Si especificas columnas: Puedes insertar solo algunos valores o cambiar el orden.

⚠️ Tipos de Datos:

Los valores deben coincidir con los tipos de datos de las columnas. Los textos van entre comillas simples, los números sin comillas.

⭐ Uso Básico del INSERT

🔍 Inserción Completa

La forma más básica de usar INSERT es proporcionando valores para todos los campos de la tabla en orden:

-- Conocer el orden de los campos SELECT * FROM prueba_insert; -- Insertar datos en orden: campo1 (INTEGER), campo2 (VARCHAR) INSERT INTO prueba_insert VALUES (1, 'valor uno');
📊 Verificación de la Inserción

Después de insertar, siempre es buena práctica verificar que los datos se guardaron correctamente:

-- Verificar la inserción SELECT * FROM prueba_insert; -- Resultado esperado: -- campo1 | campo2 -- 1 | valor uno
🔄 Múltiples Inserciones

Podemos ejecutar múltiples INSERT seguidos:

-- Primera inserción INSERT INTO prueba_insert VALUES (1, 'valor uno'); -- Segunda inserción INSERT INTO prueba_insert VALUES (2, 'valor dos'); -- Tercera inserción INSERT INTO prueba_insert VALUES (3, 'valor tres');
📋 Conocer la Estructura de la Tabla

Antes de insertar, necesitas conocer:

  • Orden de las columnas: Consulta SELECT * o doble clic en la tabla
  • Tipos de datos: INTEGER, VARCHAR, DATE, etc.
  • Longitudes: VARCHAR(50) acepta hasta 50 caracteres
  • Restricciones: NOT NULL, claves primarias, etc.
💡 Buenas Prácticas

Siempre verifica: El orden correcto de los campos y que los tipos de datos coincidan. Usa comillas simples para cadenas de texto. No uses comillas para números.

⚠️ Error Común - Longitud de Campo:

Si defines VARCHAR sin longitud, solo acepta 1 carácter. Siempre especifica la longitud: VARCHAR(50), VARCHAR(100), etc.

🎯 Especificación de Columnas

🔧 Inserción con Columnas Específicas

Cuando especificamos las columnas, ganamos flexibilidad en el orden y podemos insertar solo los campos que necesitemos:

-- Especificar columnas en orden original INSERT INTO prueba_insert (campo1, campo2) VALUES (2, 'valor dos'); -- Cambiar el orden de las columnas INSERT INTO prueba_insert (campo2, campo1) VALUES ('valor dos', 2);
🔄 Flexibilidad de Orden

Al especificar columnas, el orden de los valores debe coincidir con el orden de las columnas que especificaste, no con el orden original de la tabla:

-- Tabla original: campo1 (INTEGER), campo2 (VARCHAR) -- Pero podemos invertir el orden: INSERT INTO prueba_insert (campo2, campo1) VALUES ('nuevo valor', 5);
✅ Ventajas de Especificar Columnas
  • Orden flexible: No necesitas recordar el orden original
  • Claridad: El código es más autodocumentado
  • Inserción parcial: Solo algunos campos
  • Mantenibilidad: Si cambia la estructura, el código sigue funcionando

❌ Error de Orden

Si cambias el orden de columnas pero no el de valores, obtienes errores de tipo de datos.

✅ Orden Correcto

Los valores deben coincidir exactamente con el orden de columnas especificado.

-- ❌ INCORRECTO: Orden de valores no coincide INSERT INTO prueba_insert (campo2, campo1) VALUES (3, 'texto'); -- Error: INTEGER en VARCHAR, VARCHAR en INTEGER -- ✅ CORRECTO: Valores en orden correcto INSERT INTO prueba_insert (campo2, campo1) VALUES ('texto', 3); -- Correcto: VARCHAR en campo2, INTEGER en campo1
🔑 Regla de Oro

La secuencia de valores debe coincidir exactamente con la secuencia de columnas que especifiques, independientemente del orden original de la tabla.

⚠️ Conversiones de Tipos:

SQL puede convertir números a texto, pero NO puede convertir texto a números. Insertar 'texto' en un campo INTEGER causará error.

📊 Inserción de Valores Parciales

🎯 Inserción Parcial

No siempre necesitamos insertar valores en todos los campos. Podemos insertar solo en algunos campos específicos, dejando otros con valores NULL.

📝 Dos Formas de Inserción Parcial
  1. Con NULL explícito: Especificar NULL para campos vacíos
  2. Especificando solo columnas: Omitir campos no necesarios
-- Forma 1: NULL explícito (sin especificar columnas) INSERT INTO prueba_insert VALUES (NULL, 'solo campo2'); -- Forma 2: Especificar solo la columna que queremos INSERT INTO prueba_insert (campo2) VALUES ('nuevo valor campo2');

🔄 NULL como Estado

Sin comillas: NULL

  • Representa ausencia de valor
  • Estado especial de la base de datos
  • Se consulta con: IS NULL

📝 'NULL' como Texto

Con comillas: 'NULL'

  • Es un texto que dice "NULL"
  • Tiene valor: las letras N-U-L-L
  • Se consulta con: = 'NULL'
🔍 Diferencias en Consultas

La diferencia entre NULL y 'NULL' se aprecia al consultar:

-- Insertar NULL como estado INSERT INTO prueba_insert VALUES (NULL, 'valor'); -- Insertar 'NULL' como texto INSERT INTO prueba_insert VALUES (2, 'NULL'); -- Consultar registros con NULL verdadero SELECT * FROM prueba_insert WHERE campo1 IS NULL; -- Encuentra el primer registro -- Consultar registros con texto 'NULL' SELECT * FROM prueba_insert WHERE campo2 = 'NULL'; -- Encuentra el segundo registro
🎯 Cuándo Usar Cada Método
  • NULL explícito: Cuando conoces el orden pero algunos campos no tienen valor
  • Columnas específicas: Más claro y mantenible, especialmente con muchos campos
  • Inserción parcial: Para formularios donde no todos los campos son obligatorios
⚠️ Cuidado con NULL:

NULL sin comillas = ausencia de valor (estado especial). 'NULL' con comillas = texto que contiene las letras N-U-L-L. ¡Son completamente diferentes!

⚠️ Errores Comunes

🚨 Principales Errores al Usar INSERT

Los errores más frecuentes están relacionados con tipos de datos incompatibles y orden incorrecto de valores:

🔢 Error de Tipo de Datos

Intentar insertar texto en campo numérico o superar la longitud permitida en VARCHAR.

📐 Error de Longitud

Crear VARCHAR sin especificar tamaño o insertar texto más largo que el límite.

🔄 Error de Orden

No coincidir el orden de valores con el orden de columnas especificadas.

💭 Error de NULL

Confundir NULL (estado) con 'NULL' (texto literal).

❌ Ejemplos de Errores

Veamos los errores más comunes y cómo evitarlos:

-- ❌ ERROR 1: Tipo de datos incompatible INSERT INTO prueba_insert VALUES ('texto', 123); -- No se puede convertir 'texto' a INTEGER -- ✅ CORRECTO: INSERT INTO prueba_insert VALUES (123, 'texto'); -- INTEGER en campo1, VARCHAR en campo2 -- ❌ ERROR 2: Valor muy largo para VARCHAR INSERT INTO prueba_insert VALUES (1, 'Este texto es demasiado largo para un VARCHAR(50) y causará error'); -- ✅ CORRECTO: INSERT INTO prueba_insert VALUES (1, 'Texto adecuado'); -- Dentro del límite de 50 caracteres
🔧 Cómo Evitar Errores
  1. Verificar tipos: INTEGER para números, VARCHAR para texto
  2. Respetar longitudes: VARCHAR(50) acepta máximo 50 caracteres
  3. Orden correcto: Valores en el mismo orden que las columnas
  4. Usar comillas: Solo para texto, nunca para números
  5. NULL correcto: Sin comillas para estado nulo
-- ❌ ERROR 3: Orden incorrecto con columnas especificadas INSERT INTO prueba_insert (campo2, campo1) VALUES (5, 'texto'); -- Orden de valores no coincide con columnas -- ✅ CORRECTO: INSERT INTO prueba_insert (campo2, campo1) VALUES ('texto', 5); -- VARCHAR primero, INTEGER segundo -- ❌ ERROR 4: Confundir NULL estado con 'NULL' texto INSERT INTO prueba_insert VALUES ('NULL', 'valor'); -- Intenta insertar texto 'NULL' en INTEGER -- ✅ CORRECTO: INSERT INTO prueba_insert VALUES (NULL, 'valor'); -- NULL sin comillas para estado nulo
💡 Mensajes de Error Comunes

"Invalid column name": Nombre de columna incorrecto. "Invalid data type": Tipo de datos incompatible. "Value too long": Texto demasiado largo para el campo.

🔍 Debugging:

Cuando tengas errores, verifica: 1) Tipos de datos, 2) Orden de valores, 3) Longitud de texto, 4) Uso correcto de comillas. Los mensajes de error de SQL suelen ser específicos sobre el problema.

🎯 Conclusiones

📋 Resumen del Comando INSERT

Hemos cubierto todos los aspectos fundamentales del comando INSERT, desde la sintaxis básica hasta los errores más comunes. Este comando es esencial para alimentar nuestras tablas con datos.

1

Sintaxis Simple

INSERT INTO tabla VALUES (valores)

2

Flexibilidad

Especificar columnas para mayor control

3

Precisión

Respetar tipos de datos y longitudes

🔑 Conceptos Clave Dominados
  • Sintaxis básica: INSERT INTO tabla VALUES como estructura fundamental
  • Dos enfoques: Con y sin especificación de columnas
  • Tipos de datos: INTEGER sin comillas, VARCHAR con comillas
  • NULL vs 'NULL': Estado nulo vs texto literal
  • Orden crítico: Valores deben coincidir con columnas especificadas
🎯 Formas de Inserción

Completa: Todos los campos en orden de tabla. Específica: Solo columnas seleccionadas con flexibilidad de orden. Parcial: Algunos campos, otros quedan NULL.

⚠️ Errores a Evitar
  • Incompatibilidad de tipos: Texto en campo numérico
  • Longitud excesiva: Superar límite de VARCHAR
  • Orden incorrecto: Valores no alineados con columnas
  • Comillas incorrectas: NULL debe ir sin comillas
  • Campo sin tamaño: VARCHAR debe especificar longitud
🚀 Mejores Prácticas
  • Especifica siempre columnas: Hace el código más mantenible
  • Verifica tipos de datos: Consulta la estructura antes de insertar
  • Usa comillas correctamente: Solo para texto, nunca para números
  • Valida después de insertar: SELECT para confirmar los datos
