💡 ¿Qué Son las Funciones Agregadas?

Son funciones que realizan operaciones matemáticas sobre los datos. Como otras funciones que hemos visto (CHAR, CURRENT_DATE), nos devuelven valores, pero en este caso realizan cálculos sobre conjuntos de registros.

📚 Contenido del Módulo

  • COUNT: Conteo de registros con múltiples variantes
  • MIN/MAX: Valores mínimos y máximos
  • SUM: Suma de valores numéricos
  • AVG: Media aritmética de conjuntos
  • Relación con GROUP BY: Agregaciones por grupos
  • Funciones avanzadas: STDDEV y VAR
  • Casos prácticos: Análisis empresarial con agregados

📊 Datos de Ejemplo

Trabajaremos con la tabla "pedidos" que contiene:

  • 772 registros totales
  • 89 clientes únicos
  • IDs de pedido del 10,248 al 11,077
  • Múltiples pedidos por cliente

🚀 Introducción a las Funciones Agregadas

🎯 Definición Formal

Las funciones agregadas permiten calcular valores simples de diferentes valores de una columna. Realizan operaciones matemáticas sobre conjuntos de datos y devuelven un único resultado.

🔧 Características Fundamentales

  • Operan sobre columnas: Trabajan con los datos de una columna específica
  • Retornan un valor: Muchos registros → un resultado
  • Pueden agruparse: Diferentes resultados por grupo
  • Ignoran NULLs: (excepto COUNT(*))
  • Compatibles con WHERE: Se aplican después del filtrado
SELECT COUNT(*), MIN(columna), MAX(columna), SUM(columna), AVG(columna), STDDEV(columna), VAR(columna) FROM tabla WHERE condiciones GROUP BY columnas;
💭 Contexto en el Curso

Las funciones agregadas son el siguiente paso lógico después de dominar:

  • SELECT básico: Para obtener datos
  • WHERE: Para filtrar registros
  • ORDER BY: Para ordenar resultados
  • Funciones básicas: Como CHAR, CURRENT_DATE
📝 Preparación de Datos

Comenzaremos con una consulta básica para entender nuestros datos:

-- Consulta inicial para ver la estructura SELECT * FROM pedidos ORDER BY id_pedido ASC; -- Resultado: 772 registros -- IDs desde 10,248 hasta 11,077 -- Múltiples pedidos por cliente

🔍 Sin Funciones Agregadas

  • Vemos todos los registros individuales
  • 772 filas de resultado
  • Información dispersa
  • Difícil análisis global

📊 Con Funciones Agregadas

  • Resúmenes y totales
  • Una o pocas filas de resultado
  • Información concentrada
  • Análisis y decisiones rápidas
🎪 Analogía del Circo

Imagina que tienes 772 espectadores en un circo:

  • Sin agregados: Lista completa "Juan, María, Pedro..." (772 nombres)
  • COUNT(*): "Hay 772 espectadores"
  • AVG(edad): "La edad promedio es 35 años"
  • MAX(edad): "El más mayor tiene 89 años"

📊 Funciones Agregadas Básicas

🔢 Las Cinco Funciones Esenciales

Estas son las funciones agregadas que usarás en el 95% de los casos. Son estándar SQL y están disponibles en prácticamente todas las bases de datos.

COUNT
Cuenta el número de registros
Versátil: *, columna, DISTINCT
MIN
Valor mínimo de la columna
Funciona: Números, texto, fechas
MAX
Valor máximo de la columna
Funciona: Números, texto, fechas
SUM
Suma todos los valores
Requiere: Datos numéricos
AVG
Media aritmética
Requiere: Datos numéricos
📝 Ejemplos Básicos con la Tabla Pedidos
-- COUNT: Contar todos los registros SELECT COUNT(*) FROM pedidos; -- Resultado: 772 -- MIN: ID de pedido más bajo SELECT MIN(id_pedido) FROM pedidos; -- Resultado: 10248 -- MAX: ID de pedido más alto SELECT MAX(id_pedido) FROM pedidos; -- Resultado: 11077 -- SUM: Suma de todos los IDs (ejemplo académico) SELECT SUM(id_pedido) FROM pedidos; -- Resultado: Suma total de todos los IDs -- AVG: Promedio de los IDs SELECT AVG(id_pedido) FROM pedidos; -- Resultado: ID promedio
🔤 Funcionamiento con Diferentes Tipos de Datos

Las funciones agregadas se comportan de manera diferente según el tipo de datos:

🔢 Datos Numéricos
  • COUNT: ✅ Cuenta registros
  • MIN/MAX: ✅ Menor/mayor valor
  • SUM: ✅ Suma matemática
  • AVG: ✅ Media aritmética
🔤 Datos de Texto
  • COUNT: ✅ Cuenta registros
  • MIN/MAX: ✅ Orden alfabético (A-Z)
  • SUM: ❌ Error de tipo
  • AVG: ❌ Error de tipo
💡 Ejemplo con Diferentes Tipos de Datos
-- ✅ MIN/MAX funcionan con texto (orden alfabético) SELECT MIN(id_cliente) AS primer_cliente, MAX(id_cliente) AS ultimo_cliente FROM pedidos; -- Resultado: Primer y último cliente alfabéticamente -- ❌ SUM/AVG no funcionan con texto SELECT SUM(id_cliente) FROM pedidos; -- ERROR: Inconsistencia en tipos de datos -- Solo se permiten tipos de datos numéricos
⚠️ Errores Comunes con Tipos de Datos

SUM y AVG requieren datos numéricos. Si intentas usarlos con texto o fechas, obtendrás un error de "inconsistencia en tipos de datos".

🔢 La Función COUNT en Detalle

🎯 COUNT: La Más Versátil

COUNT es la función agregada más utilizada y tiene tres variantes importantes que se comportan de manera diferente. Es crucial entender estas diferencias para análisis precisos.

COUNT(*)
Cuenta todo
• Incluye NULLs
• Incluye duplicados
• Total de filas
COUNT(columna)
Cuenta valores
• Excluye NULLs
• Incluye duplicados
• Solo valores válidos
COUNT(DISTINCT columna)
Cuenta únicos
• Excluye NULLs
• Excluye duplicados
• Solo valores únicos
📋 Características Detalladas de COUNT
  • COUNT(*): Devuelve el número de filas total, independientemente de duplicados o NULLs
  • COUNT(columna): Cuenta registros donde esa columna NO sea NULL
  • COUNT(DISTINCT columna): Cuenta valores únicos, excluyendo NULLs y duplicados
📝 Ejemplos Prácticos con los Datos
-- COUNT(*): Total de registros SELECT COUNT(*) AS total_pedidos FROM pedidos; -- Resultado: 772 (todos los registros) -- COUNT(columna): Registros con ID_cliente válido SELECT COUNT(id_cliente) AS pedidos_con_cliente FROM pedidos; -- Resultado: 772 (no hay NULLs en este caso) -- COUNT(DISTINCT): Clientes únicos SELECT COUNT(DISTINCT id_cliente) AS clientes_unicos FROM pedidos; -- Resultado: 89 (clientes únicos)

💡 Interpretación de los Resultados

De nuestros datos podemos deducir:

  • 772 pedidos totales en la base de datos
  • 89 clientes únicos han realizado pedidos
  • Promedio: 8.7 pedidos por cliente (772 ÷ 89)
  • No hay valores NULL en id_cliente (772 = 772)
🔍 Caso Práctico: Análisis de Clientes
-- Análisis completo con una sola consulta SELECT COUNT(*) AS total_pedidos, COUNT(DISTINCT id_cliente) AS clientes_unicos, ROUND(COUNT(*)::DECIMAL / COUNT(DISTINCT id_cliente), 2) AS pedidos_por_cliente FROM pedidos; -- Resultado combinado: -- total_pedidos: 772 -- clientes_unicos: 89 -- pedidos_por_cliente: 8.67
⚠️ Importancia en Análisis Empresarial

Estas diferencias en COUNT pueden parecer detalles técnicos, pero en análisis empresarial son cruciales. Contar mal puede llevar a decisiones incorrectas sobre inventarios, personal, presupuestos, etc.

➕ Funciones Matemáticas: MIN, MAX, SUM, AVG
🧮 Operaciones Matemáticas Fundamentales

Estas cuatro funciones realizan las operaciones matemáticas más comunes en análisis de datos. Cada una tiene sus casos de uso específicos y limitaciones importantes.

MIN
Encuentra el mínimo
• Cualquier tipo de dato
• Orden: numérico, alfabético, cronológico
MAX
Encuentra el máximo
• Cualquier tipo de dato
• Orden: numérico, alfabético, cronológico
SUM
Suma matemática
• Solo datos numéricos
• Útil para totales financieros
AVG
Promedio aritmético
• Solo datos numéricos
• Útil para tendencias
📊 Ejemplos con Datos Numéricos
-- MIN: ID de pedido más bajo SELECT MIN(id_pedido) AS primer_pedido FROM pedidos; -- Resultado: 10248 -- MAX: ID de pedido más alto SELECT MAX(id_pedido) AS ultimo_pedido FROM pedidos; -- Resultado: 11077 -- SUM: Suma total de IDs (ejemplo académico) SELECT SUM(id_pedido) AS suma_ids FROM pedidos; -- Resultado: Suma de todos los IDs -- AVG: Promedio de IDs SELECT AVG(id_pedido) AS promedio_ids FROM pedidos; -- Resultado: ID promedio
🔤 Ejemplos con Datos de Texto
-- ✅ MIN/MAX funcionan con texto (orden alfabético) SELECT MIN(id_cliente) AS primer_cliente_alfabetico, MAX(id_cliente) AS ultimo_cliente_alfabetico FROM pedidos; -- Resultado: Orden A-Z y Z-A respectivamente -- ❌ SUM/AVG provocan error con texto SELECT SUM(id_cliente) FROM pedidos; -- ERROR: Inconsistencia en tipos de datos -- Solo se permiten tipos de datos numéricos
🎯 Casos de Uso Reales

En entornos empresariales, estas funciones se usan para:

  • MIN/MAX de fechas: Primera y última venta del mes
  • MIN/MAX de precios: Producto más barato y más caro
  • SUM de montos: Ingresos totales, gastos totales
  • AVG de calificaciones: Satisfacción promedio del cliente
💰 Ejemplo Empresarial Realista

Supongamos que tenemos una columna "monto_pedido":

-- Análisis financiero completo SELECT COUNT(*) AS total_pedidos, MIN(monto_pedido) AS pedido_minimo, MAX(monto_pedido) AS pedido_maximo, SUM(monto_pedido) AS ingresos_totales, AVG(monto_pedido) AS ticket_promedio FROM pedidos WHERE fecha_pedido >= '2024-01-01'; -- Resultado ejemplo: -- total_pedidos: 156 -- pedido_minimo: 15.50 -- pedido_maximo: 2,450.00 -- ingresos_totales: 89,750.30 -- ticket_promedio: 575.32
⚠️ Consideraciones Importantes
  • NULLs son ignorados: No afectan el cálculo (excepto COUNT(*))
  • División por cero: AVG de una columna sin datos válidos
  • Tipos de datos: SUM/AVG solo con números
  • Precisión decimal: AVG puede generar muchos decimales
📦 Relación Fundamental con GROUP BY
🔗 Conexión Vital

Las funciones agregadas se encuentran altamente relacionadas con el comando GROUP BY. Esta relación transforma los agregados de herramientas simples a potentes instrumentos de análisis empresarial.

📊 Sin GROUP BY
  • Resultado: Un solo valor
  • Scope: Toda la tabla
  • Información: General y sesgada
  • Utilidad: Totales globales
📦 Con GROUP BY
  • Resultado: Un valor por grupo
  • Scope: Cada grupo independiente
  • Información: Detallada y específica
  • Utilidad: Análisis comparativo
🎯 ¿Por Qué Es Tan Importante?

Calcular valores a nivel de toda la tabla nos da información muy sesgada. Es como una empresa multinacional que solo mira beneficios globales sin saber de dónde vienen las ganancias o pérdidas.

📈 Ejemplo: Análisis Global vs Por Cliente
-- SIN GROUP BY: Información sesgada SELECT COUNT(*) AS total_pedidos, AVG(id_pedido) AS promedio_global FROM pedidos; -- Resultado: 772 pedidos, promedio general -- ¿Pero cuáles clientes son más importantes? -- CON GROUP BY: Información detallada y útil SELECT id_cliente, COUNT(*) AS pedidos_cliente, MIN(id_pedido) AS primer_pedido, MAX(id_pedido) AS ultimo_pedido FROM pedidos GROUP BY id_cliente ORDER BY pedidos_cliente DESC; -- Resultado: Cada cliente con sus estadísticas
💼 Resultados del Ejemplo Práctico
-- Resultados ejemplo con GROUP BY: -- Cliente A: 10 pedidos (excelente cliente) -- Cliente B: 9 pedidos (buen cliente) -- Cliente C: 6 pedidos (cliente regular) -- Cliente D: 1 pedido (cliente ocasional) -- Ahora podemos tomar decisiones: -- • Cuidar a clientes A y B (ofertas especiales) -- • Ignorar o dar menos prioridad a cliente D -- • Intentar convertir C en cliente premium
🎯 Estrategias Empresariales Basadas en Agregados

Con funciones agregadas y GROUP BY puedes implementar:

  • Segmentación de clientes: Premium, regular, ocasional
  • Análisis de productos: Más vendidos, menos vendidos
  • Gestión de recursos: Dónde invertir tiempo y dinero
  • Detección de patrones: Tendencias por región, época, etc.
🔍 Más Ejemplos con GROUP BY
-- Análisis por cliente: pedidos y rango de IDs SELECT id_cliente, COUNT(*) AS total_pedidos, MIN(id_pedido) AS id_minimo, MAX(id_pedido) AS id_maximo, SUM(id_pedido) AS suma_ids FROM pedidos GROUP BY id_cliente HAVING COUNT(*) > 5 -- Solo clientes con más de 5 pedidos ORDER BY total_pedidos DESC; -- Resultado: Solo clientes importantes (>5 pedidos) -- Ordenados por importancia (más pedidos primero)
🚀 Escalabilidad del Análisis

Este mismo patrón se puede aplicar a cualquier nivel:

  • GROUP BY región: Ventas por zona geográfica
  • GROUP BY producto: Performance de cada producto
  • GROUP BY mes: Tendencias temporales
  • GROUP BY vendedor: Performance del equipo
💡 Ejemplos Prácticos y Casos de Uso
🎪 De la Teoría a la Práctica

Ahora que entendemos las funciones agregadas individualmente y su relación con GROUP BY, veamos casos prácticos reales que encontrarás en entornos empresariales.

📊 Caso 1: Dashboard Ejecutivo

Necesitas crear un resumen ejecutivo con métricas clave:

-- Dashboard ejecutivo en una sola consulta SELECT 'Resumen Ejecutivo' AS reporte, COUNT(*) AS total_pedidos, COUNT(DISTINCT id_cliente) AS clientes_activos, MIN(id_pedido) AS primer_pedido, MAX(id_pedido) AS ultimo_pedido, ROUND(AVG(id_pedido), 0) AS pedido_promedio FROM pedidos; -- Resultado ejemplo: -- Resumen Ejecutivo -- • 772 pedidos procesados -- • 89 clientes activos -- • Rango de pedidos: 10248 - 11077 -- • ID promedio: 10663
🏆 Caso 2: Top 10 Clientes Más Importantes
-- Identificar clientes VIP por número de pedidos SELECT id_cliente, COUNT(*) AS total_pedidos, MIN(id_pedido) AS primer_pedido, MAX(id_pedido) AS ultimo_pedido, CASE WHEN COUNT(*) >= 15 THEN 'VIP' WHEN COUNT(*) >= 10 THEN 'Premium' WHEN COUNT(*) >= 5 THEN 'Regular' ELSE 'Ocasional' END AS categoria_cliente FROM pedidos GROUP BY id_cliente ORDER BY total_pedidos DESC LIMIT 10; -- Resultado: Top 10 con categorización automática
📈 Caso 3: Análisis de Distribución de Clientes
-- Estadísticas de segmentación de clientes SELECT CASE WHEN pedidos_por_cliente >= 15 THEN 'VIP (15+)' WHEN pedidos_por_cliente >= 10 THEN 'Premium (10-14)' WHEN pedidos_por_cliente >= 5 THEN 'Regular (5-9)' ELSE 'Ocasional (1-4)' END AS categoria, COUNT(*) AS cantidad_clientes FROM ( SELECT id_cliente, COUNT(*) AS pedidos_por_cliente FROM pedidos GROUP BY id_cliente ) AS cliente_stats GROUP BY CASE WHEN pedidos_por_cliente >= 15 THEN 'VIP (15+)' WHEN pedidos_por_cliente >= 10 THEN 'Premium (10-14)' WHEN pedidos_por_cliente >= 5 THEN 'Regular (5-9)' ELSE 'Ocasional (1-4)' END ORDER BY cantidad_clientes DESC; -- Resultado ejemplo: -- Ocasional (1-4): 45 clientes -- Regular (5-9): 28 clientes -- Premium (10-14): 12 clientes -- VIP (15+): 4 clientes
💼 Decisiones Empresariales Basadas en Datos

Con estos análisis, la empresa puede:

  • Recursos limitados: Enfocar atención en 4 clientes VIP
  • Estrategias diferenciadas: Ofertas personalizadas por segmento
  • Conversión: Intentar mover clientes de Regular a Premium
  • Retención: Evitar que Premium bajen a Regular
🔄 Caso 4: Comparativa Temporal

Si tuviéramos fecha de pedidos, podríamos hacer análisis temporales:

-- Ejemplo hipotético con fechas SELECT EXTRACT(YEAR FROM fecha_pedido) AS año, EXTRACT(MONTH FROM fecha_pedido) AS mes, COUNT(*) AS pedidos_del_mes, COUNT(DISTINCT id_cliente) AS clientes_activos_mes, AVG(monto_pedido) AS ticket_promedio_mes FROM pedidos WHERE fecha_pedido >= '2024-01-01' GROUP BY EXTRACT(YEAR FROM fecha_pedido), EXTRACT(MONTH FROM fecha_pedido) ORDER BY año, mes; -- Resultado: Tendencias mensuales de negocio

🔬 Funciones Avanzadas: STDDEV y VAR
📊 Funciones Estadísticas Especializadas

SAP HANA incluye funciones estadísticas más avanzadas como STDDEV (desviación estándar) y VAR (varianza). Aunque no son estándar SQL y su uso es menos común, están disponibles para análisis estadísticos especializados.

STDDEV
Desviación Estándar
• Mide dispersión de datos
• No estándar SQL
• Uso especializado
VAR
Varianza
• Mide variabilidad
• No estándar SQL
• Análisis estadístico
⚠️ Limitaciones del Instructor

Transparencia total: El instructor admite que no domina estas funciones estadísticas. Están incluidas en la documentación pero no son de uso habitual en el desarrollo típico de aplicaciones empresariales.

🎯 ¿Cuándo Podrías Necesitarlas?

Estas funciones son útiles en contextos muy específicos:

  • Análisis científico: Investigación con datos numéricos
  • Control de calidad: Medición de variabilidad en procesos
  • Análisis financiero: Volatilidad de precios o rendimientos
  • Estudios estadísticos: Distribución de datos
📝 Sintaxis Básica (Sin Profundizar)
-- Sintaxis básica de funciones avanzadas SELECT STDDEV(id_pedido) AS desviacion_estandar, VAR(id_pedido) AS varianza FROM pedidos; -- También funcionan con GROUP BY SELECT id_cliente, COUNT(*) AS pedidos, AVG(id_pedido) AS promedio, STDDEV(id_pedido) AS desviacion FROM pedidos GROUP BY id_cliente HAVING COUNT(*) > 5;
🏢 Realidad Empresarial

En la práctica empresarial real:

  • 95% de casos: COUNT, MIN, MAX, SUM, AVG son suficientes
  • 4% de casos: Funciones de fecha y texto especializadas
  • 1% de casos: Funciones estadísticas como STDDEV/VAR
