💡 Definición Principal

Una expresión es una cláusula que puede ser evaluada para retornar un valor. Por ejemplo: 1 = 1 (verdadero), 1 ≠ 2 (verdadero), o 1 = 2 (falso).

🚀 Introducción a las Expresiones

🌟 Continuando con HAN SQL

Continuamos avanzando con más elementos del lenguaje HAN SQL. Hoy nos enfocamos en las expresiones, un concepto fundamental que amplía significativamente nuestras capacidades de manipulación de datos.

📖 ¿Qué es una Expresión?

Como definición genérica, podríamos decir que una expresión es una cláusula que puede ser evaluada para retornar un valor. Es un concepto simple pero muy poderoso en SQL.

📝 Ejemplos Básicos de Expresiones

Para entender el concepto, veamos algunos ejemplos fundamentales:

  • 1 = 1 → Sabemos que esto es cierto (retorna TRUE)
  • 1 ≠ 2 → También es cierto (retorna TRUE)
  • 1 = 2 → Esto es falso (retorna FALSE)

Todos estos son ejemplos de expresiones, independientemente de su resultado.

🎯 Importancia de las Expresiones

Las expresiones nos permiten:

  • Evaluar condiciones: Tomar decisiones basadas en datos
  • Transformar valores: Convertir datos de un formato a otro
  • Crear lógica condicional: Comportamiento dinámico en consultas
  • Manipular datos: Aplicar funciones y operaciones
🔍 Contexto en el Curso

Este tema nos introduce conceptos que veremos más adelante:

  • Funciones avanzadas: Manipulación de fechas, textos y números
  • Lógica condicional: IF statements y casos complejos
  • Procedimientos: Encapsulación de código reutilizable
  • Expresiones complejas: Combinaciones de múltiples operaciones
💡 Concepto Clave

Una expresión siempre retorna un valor. Puede ser verdadero/falso, un número, una fecha, texto, o NULL. Lo importante es que el resultado puede ser utilizado en otras partes de la consulta.

📖 Definición y Ejemplos de Expresiones
📚 Definición Formal

Una expresión es una cláusula que puede ser evaluada para retornar un valor. Es la unidad básica de evaluación en SQL que produce un resultado específico.

🔍 Características de las Expresiones
  • Evaluables: Pueden ser procesadas por el motor de base de datos
  • Retornan valores: Siempre producen un resultado
  • Reutilizables: Pueden usarse en diferentes contextos
  • Combinables: Se pueden combinar para crear expresiones más complejas
📝 Ejemplos Fundamentales

Veamos los ejemplos más básicos que nos ayudan a entender el concepto:

-- Expresiones de comparación 1 = 1 -- Retorna TRUE (verdadero) 1 != 2 -- Retorna TRUE (verdadero) 1 = 2 -- Retorna FALSE (falso) -- Expresiones aritméticas 5 + 3 -- Retorna 8 10 / 2 -- Retorna 5 2 * 4 -- Retorna 8
⚡ Tipos de Valores Retornados

Las expresiones pueden retornar diferentes tipos de valores:

  • Booleanos: TRUE/FALSE (verdadero/falso)
  • Números: Enteros, decimales, etc.
  • Texto: Cadenas de caracteres
  • Fechas: Valores temporales
  • NULL: Valor nulo o ausente
🎯 Contextos de Uso

Las expresiones se pueden usar en:

  • Cláusula SELECT: Para calcular campos derivados
  • Cláusula WHERE: Para filtrar registros
  • Cláusula HAVING: Para filtrar grupos
  • Cláusula ORDER BY: Para ordenar resultados
  • Funciones: Como parámetros de entrada
✅ Ejemplo de Evaluación
Expresión: 1 = 1
Evaluación: ¿Es 1 igual a 1?
Resultado: TRUE
🔧 Expresiones en SQL

En el contexto de SQL, las expresiones son fundamentales para:

  • Transformación de datos: Cambiar formato o tipo
  • Cálculos: Operaciones matemáticas
  • Lógica condicional: Decisiones basadas en datos
  • Validaciones: Verificar condiciones
💡 Punto Clave

Lo importante no es si el resultado de la expresión es verdadero o falso, sino que puede ser evaluada y retorna un valor. Incluso 1 = 2 (que es falso) sigue siendo una expresión válida.

⚡ Expresiones Condicionales

Son expresiones que permiten seleccionar un valor según una condición sin usar procedimientos. Esto es muy importante porque nos da flexibilidad sin la complejidad de crear procedimientos separados.

🔧 ¿Qué es un Procedimiento?

Un procedimiento es un conjunto de líneas de código con un nombre que podemos llamar o reutilizar tantas veces como queramos o necesitemos.

-- Concepto visual de procedimiento Mi Código Principal: Línea 1 Línea 2 CALL mi_procedimiento() -- Llama al procedimiento Línea 4 Línea 5 mi_procedimiento: -- Código encapsulado Línea A Línea B Línea C RETURN -- Vuelve al código principal
🎯 Ventaja de las Expresiones Condicionales

Las expresiones condicionales nos permiten hacer lógica condicional directamente en la consulta, sin necesidad de:

  • Crear procedimientos separados
  • Escribir código adicional
  • Gestionar llamadas a funciones
  • Mantener código en múltiples lugares
📝 Ejemplos de Expresiones Condicionales
  • Comando CASE: El más utilizado en SQL
  • Comando IF: Para lógica simple (lo veremos más adelante)
  • Funciones condicionales: NULLIF, COALESCE, etc.
🛠️ Expresiones de Función

Son funciones del lenguaje HAN SQL que se pueden usar como expresiones. Estas funciones realizan operaciones específicas y retornan valores que pueden ser utilizados en nuestras consultas.

📅 Ejemplo: CURRENT_DATE

La función CURRENT_DATE es un ejemplo perfecto de expresión de función. Retorna la fecha actual del sistema y puede ser utilizada en cualquier contexto donde se necesite una expresión.

-- Ejemplo de expresión de función SELECT CURRENT_DATE FROM DUMMY; -- Resultado: 2024-12-24 (fecha actual del sistema)
💡 Comparación de Tipos

Expresiones Condicionales: Evalúan condiciones y retornan valores basados en esas condiciones.

Expresiones de Función: Ejecutan operaciones específicas y retornan sus resultados.

Ambas son expresiones: Porque pueden ser evaluadas y retornan valores.

⚡ Expresiones Condicionales
🎯 Concepto Principal

Las expresiones condicionales son herramientas que nos permiten seleccionar un valor según una condición sin usar procedimientos. Esto significa que podemos implementar lógica directamente en nuestras consultas SQL.

🔧 ¿Por Qué "Sin Procedimientos"?

Esta característica es importante porque:

  • Simplicidad: No necesitamos crear código separado
  • Rendimiento: Evaluación directa en la consulta
  • Mantenimiento: Todo el código está en un lugar
  • Portabilidad: Funciona sin dependencias externas
📊 Visualización del Concepto de Procedimiento

Para entender mejor por qué es ventajoso no usar procedimientos, veamos cómo funciona conceptualmente un procedimiento:

-- Código principal Línea 1: Obtener datos Línea 2: Procesar información Línea 3: CALL obtener_nombre_mes() -- Salto a procedimiento Línea 4: Continuar procesamiento Línea 5: Finalizar -- Procedimiento separado (en otra ubicación) PROCEDURE obtener_nombre_mes: Evaluar mes actual Retornar nombre correspondiente RETURN resultado -- Vuelve a línea 4
✨ Ventajas de las Expresiones Condicionales
  • Evaluación inmediata: El resultado se calcula directamente
  • Código cohesivo: Toda la lógica está en la consulta
  • Fácil debugging: Puedes ver toda la lógica de una vez
  • Menos dependencias: No relies en código externo
  • Mejor rendimiento: Sin overhead de llamadas a funciones
📝 Herramientas Principales

Las principales herramientas para expresiones condicionales en HAN SQL son:

  • CASE: La más utilizada y versátil
  • IF: Para lógica simple (tema futuro)
  • NULLIF: Para manejar valores nulos
  • COALESCE: Para seleccionar el primer valor no nulo
🌟 El Comando CASE

El comando CASE es la herramienta principal para expresiones condicionales en SQL. Nos permite evaluar múltiples condiciones y retornar diferentes valores según cada caso.

🔍 Casos de Uso Comunes
  • Transformación de códigos: Convertir números en texto descriptivo
  • Categorización: Agrupar valores en rangos o categorías
  • Formateo condicional: Mostrar datos según el contexto
  • Cálculos condicionales: Aplicar diferentes fórmulas según condiciones
  • Validación de datos: Verificar y corregir inconsistencias

📝 Comando CASE
🌟 La Herramienta Principal

El comando CASE es la herramienta más importante para expresiones condicionales en SQL. Nos permite evaluar múltiples condiciones y retornar diferentes valores según cada caso específico.

📋 Sintaxis del Comando CASE

La estructura básica del comando CASE sigue un patrón específico y ordenado:

-- Sintaxis básica del comando CASE CASE variable WHEN valor1 THEN resultado1 WHEN valor2 THEN resultado2 WHEN valor3 THEN resultado3 ELSE resultado_por_defecto END
🔧 Componentes del CASE
  • CASE variable: Inicia la expresión e indica qué valor evaluar
  • WHEN valor: Especifica cada condición a verificar
  • THEN resultado: Define qué retornar si se cumple la condición
  • ELSE: Valor por defecto si no se cumple ninguna condición (opcional)
  • END: Cierra la expresión CASE
⚡ Funcionamiento

El CASE evalúa la variable contra cada WHEN en orden:

  1. Evalúa si variable = valor1 → Si es cierto, retorna resultado1
  2. Si no, evalúa si variable = valor2 → Si es cierto, retorna resultado2
  3. Continúa hasta encontrar una coincidencia
  4. Si no encuentra ninguna, retorna el valor del ELSE
  5. Si no hay ELSE, retorna NULL
📝 Ejemplo Simple

Veamos un ejemplo básico para entender el funcionamiento:

-- Ejemplo: Convertir código de estado civil a texto SELECT CASE estado_civil WHEN 'S' THEN 'Soltero' WHEN 'C' THEN 'Casado' WHEN 'D' THEN 'Divorciado' ELSE 'Estado Desconocido' END AS estado_descripcion FROM personas;
🔍 Características Importantes
  • Evaluación secuencial: Se detiene en la primera coincidencia
  • Tipos consistentes: Todos los THEN deben retornar el mismo tipo de dato
  • ELSE opcional: Pero recomendado para evitar NULLs inesperados
  • Anidable: Puedes poner un CASE dentro de otro CASE
  • Usable en cualquier lugar: SELECT, WHERE, ORDER BY, etc.
⚠️ Consejos de Sintaxis

Importante para principiantes: Presta mucha atención a la puntuación en el comando CASE. Los errores de comas y puntos y comas son muy comunes y pueden ser difíciles de detectar.

  • No hay comas entre WHEN: Cada WHEN va en su propia línea sin coma
  • Sin punto y coma interno: El CASE es una sola expresión
  • END obligatorio: Siempre cierra con END
  • Consistencia de tipos: Todos los THEN deben ser del mismo tipo

🛠️ Funciones y Herramientas
🎯 Funciones para el Ejemplo

Para crear un ejemplo completo y útil del comando CASE, necesitamos introducir algunas funciones y herramientas avanzadas. Aunque las estudiaremos en detalle más adelante, las presentamos ahora para entender mejor las expresiones.

📅 Función CURRENT_DATE

CURRENT_DATE es una función del sistema que nos devuelve la fecha actual del sistema. Es muy útil para trabajar con fechas dinámicas.

-- Uso básico de CURRENT_DATE SELECT CURRENT_DATE FROM DUMMY; -- Resultado: 2024-12-24 (fecha actual del sistema)
🔧 Puntos Importantes sobre CURRENT_DATE
  • Sin parámetros: No necesita argumentos
  • Valor dinámico: Cambia automáticamente cada día
  • Formato estándar: Retorna en formato YYYY-MM-DD
  • Función del sistema: Obtiene la fecha del servidor
📝 Tabla DUMMY

DUMMY es una tabla especial del sistema que nos sirve para hacer pruebas y mostrar datos por pantalla. Es una tabla vacía pero muy útil.

-- Verificar contenido de DUMMY SELECT * FROM DUMMY; -- Resultado: tabla vacía, pero permite ejecutar funciones
🎯 Características de DUMMY
  • Tabla del sistema: Siempre disponible
  • Vacía: No contiene datos reales
  • Para pruebas: Ideal para probar funciones
  • Sin overhead: No impacta rendimiento
  • Estándar: Usada frecuentemente en ejemplos
🔄 Función TOCHAR

TOCHAR es una función que transforma el tipo de dato del valor que le pasemos, convirtiéndolo a tipo carácter (texto).

🔧 Funcionamiento de TOCHAR
  • Número → Texto: Convierte números a cadenas de caracteres
  • Fecha → Texto: Convierte fechas a formato texto
  • Con máscaras: Permite especificar el formato de salida
  • Flexible: Múltiples opciones de formateo
-- Ejemplos de TOCHAR SELECT TOCHAR(123) FROM DUMMY; -- '123' (como texto) SELECT TOCHAR(CURRENT_DATE) FROM DUMMY; -- '2024-12-24' -- Con máscara para obtener mes abreviado SELECT TOCHAR(CURRENT_DATE, 'MON') FROM DUMMY; -- 'DEC'
🎨 Máscaras de Formato

Las máscaras sirven para especificar cómo queremos que se muestre la salida:

  • 'MON': Mes abreviado (JAN, FEB, MAR...)
  • 'MONTH': Mes completo (JANUARY, FEBRUARY...)
  • 'DD': Día del mes (01, 02, 03...)
  • 'YYYY': Año completo (2024, 2025...)
🌟 Familia de Funciones de Conversión

TOCHAR pertenece a una familia de funciones de conversión:

  • TOCHAR: Convierte a tipo carácter/texto
  • TO_INTEGER: Convierte a números enteros
  • TO_DATE: Convierte a tipo fecha
  • TO_DECIMAL: Convierte a números decimales
💡 Preparación para el Ejemplo

Con estas herramientas podemos crear un ejemplo práctico que combine CASE + CURRENT_DATE + TOCHAR + DUMMY para mostrar el nombre del mes actual en español.

💻 Ejemplo Práctico Completo
🎯 Objetivo del Ejemplo

Vamos a crear una consulta que nos indicará el nombre del mes en el que nos encontramos, combinando todas las herramientas que hemos aprendido: CASE, CURRENT_DATE, TOCHAR y la tabla DUMMY.

🔧 Construcción Paso a Paso

Construiremos la consulta de forma incremental para entender cada parte:

-- Paso 1: Obtener el mes actual abreviado SELECT TOCHAR(CURRENT_DATE, 'MON') FROM DUMMY; -- Resultado: 'DEC' (para diciembre)
-- Paso 2: Consulta completa con CASE SELECT CASE TOCHAR(CURRENT_DATE, 'MON') WHEN 'JAN' THEN 'Enero' WHEN 'FEB' THEN 'Febrero' WHEN 'MAR' THEN 'Marzo' WHEN 'APR' THEN 'Abril' WHEN 'MAY' THEN 'Mayo' WHEN 'JUN' THEN 'Junio' WHEN 'JUL' THEN 'Julio' WHEN 'AUG' THEN 'Agosto' WHEN 'SEP' THEN 'Septiembre' WHEN 'OCT' THEN 'Octubre' WHEN 'NOV' THEN 'Noviembre' WHEN 'DEC' THEN 'Diciembre' ELSE 'Mes desconocido' END AS mes_actual FROM DUMMY;
✅ Resultado de la Consulta
mes_actual
----------
Diciembre

Estamos en diciembre de 2024, por lo que muestra "Diciembre"

🔍 Análisis del Funcionamiento
  1. CURRENT_DATE obtiene la fecha actual del sistema
  2. TOCHAR(..., 'MON') extrae el mes abreviado ('DEC')
  3. CASE evalúa el valor 'DEC' contra cada WHEN
  4. Encuentra coincidencia con WHEN 'DEC'
  5. Retorna 'Diciembre' como resultado
🧪 Probando con Otras Fechas

Podemos probar la lógica con fechas específicas:

-- Probando con una fecha específica (marzo) SELECT CASE TOCHAR('2024-03-12', 'MON') WHEN 'JAN' THEN 'Enero' WHEN 'FEB' THEN 'Febrero' WHEN 'MAR' THEN 'Marzo' -- ... resto de meses ... ELSE 'Mes desconocido' END AS mes_especifico FROM DUMMY; -- Resultado: 'Marzo'
⚠️ Consejos de Sintaxis Aplicados

Observa los detalles importantes de sintaxis en el ejemplo:

  • Sin comas entre WHEN: Cada línea WHEN va independiente
  • Saltos de línea permitidos: La consulta puede ocupar múltiples líneas
  • ELSE incluido: Para manejar casos inesperados
  • END obligatorio: Cierra correctamente el CASE
  • AS para alias: Nombra el resultado como 'mes_actual'
🎯 Verificación del Resultado

Para verificar que funciona correctamente, podemos ver el valor intermedio:

-- Verificar el valor que evalúa el CASE SELECT TOCHAR(CURRENT_DATE, 'MON') FROM DUMMY; -- Resultado: 'DEC' -- Este valor 'DEC' es lo que evalúa el CASE -- Coincide con WHEN 'DEC' THEN 'Diciembre'

Este ejemplo demuestra el poder de las expresiones: combinamos una función de fecha, conversión de tipos y lógica condicional en una sola consulta para obtener un resultado útil y legible.
