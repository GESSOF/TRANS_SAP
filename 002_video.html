🎯 Concepto Fundamental

La arquitectura de SAP HANA es fundamental para comprender cómo funciona este sistema, especialmente importante para el examen de certificación.

Un entorno de SAP HANA está compuesto por muchos componentes, y los clientes no requieren todos ellos.

🔑 Concepto Clave

SAP HANA no es un producto monolítico, sino un sistema modular con una base común y diferentes añadidos según las necesidades específicas.

📌 Puntos Importantes para Certificación

  • Sistema modular: Base común + componentes según necesidades
  • Flexibilidad: Desde básico hasta empresarial complejo
  • Escalabilidad: Puede crecer según requerimientos
  • Integración: Múltiples herramientas y motores

🎓 Importancia Académica

Este contenido está diseñado especialmente para:

  • Comprensión del curso: Base para temas posteriores
  • Examen de certificación: Preguntas frecuentes sobre arquitectura
  • Trabajo profesional: Entender qué componentes usar

📦 Ediciones de SAP HANA

🎯 Estrategia de Ediciones

SAP ofrece diferentes ediciones para adaptarse a distintos tipos de usuarios y necesidades empresariales, desde estudiantes hasta grandes corporaciones.

📊 SAP HANA Standard Edition

La versión más básica del sistema

🔧 Características

  • Núcleo de la base de datos: Funcionalidad esencial
  • Servicios de integración incluidos: Conectividad básica
  • Aplicaciones necesarias: Para trabajar con la base de datos
  • Preparada para expansión: Añadir componentes mediante instalación

💡 Uso Recomendado

Es la base sobre la cual se puede construir un sistema personalizado añadiendo módulos según las necesidades del negocio.

🏢 SAP HANA Enterprise Edition

La versión más completa disponible

✅ Características

  • Todo incluido: Contiene todo el contenido de la edición estándar
  • Funcionalidades adicionales: Muchas características integradas
  • Desarrollo optimizado: Mejor preparada para aplicaciones sofisticadas
  • Sin complicaciones: Ideal para basar todo el sistema sobre HANA

🎯 Uso Recomendado

Ideal para empresas que quieren basar todo su sistema sobre HANA sin complicaciones de configuración.

🎓 SAP HANA Express Edition

Totalmente gratuita - Diseñada para estudiantes y aprendizaje

✅ Características

  • Gratuita al 100%: Sin costo para uso educativo
  • Requisitos reducidos: Puede instalarse en equipos modestos
  • Descarga directa: Disponible desde el portal SAP
  • Opción cloud: Despliegue en proveedores certificados

📊 Límites de Uso

  • Hasta 32 GB: De memoria RAM sin costo
  • Ampliación a 128 GB: Con licencia
  • Más memoria: Requiere licencias Standard o Enterprise

🎯 Ideal Para

  • Estudiantes y aprendizaje
  • Desarrollo y pruebas
  • Proyectos pequeños
  • Evaluación de la tecnología

🔧 Componentes del Sistema Básico

⚙️ Componentes Universales

Los siguientes componentes están presentes en todas las ediciones de SAP HANA y forman el núcleo del sistema.

💾 Base de Datos

Componente fundamental que actúa como capa de persistencia:

  • Tablas físicas: Almacenadas en disco
  • Tablas virtuales: Smart Data Access (SDA)
  • Dual storage: Por filas y por columnas
  • Gestión de memoria: Optimización automática

⚡ Motores de Procesamiento

Proporcionan capacidad de procesamiento en memoria:

  • Motor de cálculo: Para vistas calculadas
  • Motor SQL: Comunicación con base de datos
  • Motor MDX: Consultas multidimensionales
  • Optimización: Procesamiento paralelo

🌐 Motor XS/XSA

Servidor de aplicaciones integrado en la base de datos:

  • XS: Versión original (HANA 1.0)
  • XSA: Extended Application Service Advanced
  • Herramientas: Desarrollo y depuración
  • Independencia: HANA como plataforma completa

📚 PREGUNTA DE EXAMEN

XSA se basa en el estándar abierto Cloud Foundry

💻 Interfaces de Usuario

🔧 Herramientas de Desarrollo y Administración

SAP HANA ofrece diferentes interfaces según el tipo de usuario y las tareas a realizar.

🖥️ SAP HANA Studio

⚠️ En Desuso

  • Cliente pesado: Windows, Mac y Linux
  • Basado en Eclipse: Con plugins específicos
  • Quedando obsoleto: Solo para tareas específicas
  • Uso limitado: Solo desarrollo con motor XS
  • Administración: Algunas tareas específicas

🌐 Web IDE for SAP HANA

✅ Futuro del Desarrollo

  • Interfaz web: Para motor XSA
  • Desarrolladores: Y modeladores principalmente
  • Sin administración: No permite tareas de admin
  • Moderno: Futuro de desarrollo en HANA

📚 PREGUNTA DE EXAMEN

Web IDE solo para XSA, HANA Studio solo para XS

🎛️ SAP HANA Cockpit

🔧 Administración Web

  • Interfaz web: Para administración completa
  • Basado en SAP Fiori: UX moderna
  • Monitoreo: Del sistema en tiempo real
  • Centralizado: Herramientas de administración

🔄 Herramientas de Aprovisionamiento de Datos

📊 Enterprise Information Management (EIM)

Conjunto de herramientas para gestión y calidad de datos en el ecosistema SAP HANA.

🔧 SDI y SDQ

🎯 Smart Data Integration & Quality

  • SDI (Smart Data Integration): Herramienta ETL avanzada
  • SDQ (Smart Data Quality): Limpieza y validación de datos
  • EIM: Parte del Enterprise Information Management
  • Integración: Con múltiples fuentes de datos

📚 PREGUNTA DE EXAMEN

Recordar las siglas SDI y SDQ

🚀 SLT

📡 SAP Landscape Transformation

  • Replication Server: Para sistemas SAP
  • Módulo ETL: Especializado en ecosistema SAP
  • Servidor dedicado: Preferiblemente independiente
  • Integración: Trabaja con SAP Data Services

📡 Streaming Analytics

⚡ Datos en Tiempo Real

  • Transmisión continua: Desde múltiples canales
  • IoT Integration: Sensores, registros, máquinas
  • Análisis inmediato: Procesamiento en tiempo real
  • Escalabilidad: Miles de fuentes simultáneas

🔄 Remote Data Sync (RDS)

📱 Sincronización Remota

  • Dispositivos offline: No conectados continuamente
  • Sincronización: Al reconectarse
  • Móviles: Tablets, smartphones
  • Trabajo remoto: Equipos en campo

🌊 Flujo de Datos Moderno

Estas herramientas permiten un flujo continuo y optimizado de datos desde múltiples fuentes hacia SAP HANA, garantizando calidad y disponibilidad en tiempo real.

➕ Componentes Adicionales

🏗️ Data Warehousing Foundation

Proporciona funcionalidad y componentes adicionales requeridos para crear un almacén de datos nativo usando solo SAP HANA.

🔄 Enfoque

Similar a adaptar un Business Warehouse a una plataforma HANA, pero diseñado específicamente para aprovechar todas las capacidades nativas del sistema.

🌡️ Dynamic Tiering

Proporciona una capa basada en disco para complementar el nivel de memoria existente.

🎯 Concepto de Temperatura de Datos

  • Datos calientes: Alta frecuencia de acceso → Memoria RAM
  • Datos fríos: Baja frecuencia de acceso → Disco duro
  • Optimización económica: Ahorro sin sacrificar velocidad
  • Gestión automática: Movimiento dinámico según uso

✅ Beneficios

  • Ahorro de costos: Menos memoria RAM requerida
  • Sin pérdida de rendimiento: Para consultas críticas
  • Escalabilidad: Manejo de volúmenes masivos
  • Transparencia: Invisible para aplicaciones

🏗️ Arquitecturas de Implementación

🏢 Stack de Tres Capas

La arquitectura tradicional empresarial con separación clara de responsabilidades.

🖥️ Capa Cliente - Interfaces de Usuario
⚙️ Capa Servidor de Aplicaciones - SAP NetWeaver (Lógica Empresarial)
💾 Capa de Base de Datos - SAP HANA (Persistencia y Procesamiento)

📚 CONCEPTO DE EXAMEN

Push Down: Siempre que sea posible, los procesamientos deben realizarse en la capa de base de datos para aprovechar la aceleración de HANA.

📊 Stack de Dos Capas

Arquitectura simplificada que elimina capas intermedias para máximo rendimiento.

✅ Para Business Intelligence

  • Sin servidor intermedio: Elimina la capa del servidor de aplicaciones
  • Comunicación directa: Entre herramientas BI y HANA
  • Protocolos estándar: JDBC, ODBC, ODBO, BICS
  • Adquisición de datos: Mediante EIM o herramientas externas

🌐 Con Motor XS/XSA

La base de datos actúa también como servidor de aplicaciones, proporcionando una plataforma completa e integrada.

🏢 Tres Capas

Separación tradicional

• Cliente
• Servidor App
• Base de Datos

Máximo rendimiento

🚀 Evolución XS → XSA

📈 Evolución Tecnológica

La transición de XS a XSA representa un salto significativo en capacidades y arquitectura del servidor de aplicaciones integrado.

🏗️ Motor XS (HANA 1.0)

⚠️ Limitaciones
  • Servidor ligero: Capacidades básicas
  • No escalable: No para aplicaciones empresariales
  • JavaScript y HTML: Tecnologías limitadas
  • Servidor web básico: Funcionalidades mínimas

🚀 Motor XSA (HANA 2.0)

✅ Capacidades Avanzadas
  • Mucho más potente: Aplicaciones empresariales
  • Cloud Foundry: Estándar abierto
  • Independencia: De infraestructura
  • Flexibilidad: On-premises o cloud

🔧 Características Técnicas de XSA

🎯 Ventajas Clave

  • Basado en Cloud Foundry: Estándar abierto de la industria
  • Aplicaciones independientes: De la infraestructura subyacente
  • Implementación flexible: On-premises o cloud sin cambios
  • Escalabilidad empresarial: Capacidad para aplicaciones críticas

📚 PREGUNTAS DE EXAMEN IMPORTANTES

  • XSA se basa en Cloud Foundry
  • Web IDE solo para XSA, HANA Studio solo para XS
  • Las aplicaciones XS no son compatibles con XSA

⚠️ Incompatibilidad

Las aplicaciones XS no son compatibles con XSA - Se requiere migración y rediseño para aprovechar las nuevas capacidades.

🎯 Resumen de Preguntas de Examen

📋 Lista de Verificación

  • ✅ XSA y Cloud Foundry: Estándar abierto
  • ✅ Herramientas específicas: Web IDE (XSA) vs HANA Studio (XS)
  • ✅ SDI y SDQ: Smart Data Integration y Quality
  • ✅ Push Down: Procesamiento en capa de base de datos
  • ✅ Arquitecturas: 2 capas vs 3 capas

🎯 Conclusión

La arquitectura de SAP HANA es modular y flexible, permitiendo implementaciones desde básicas hasta empresariales complejas. La comprensión de estos componentes es fundamental tanto para el trabajo práctico como para la certificación.
