Curso de Modelado en SAP HANA - Comprensión de la Arquitectura del Sistema

Introducción a la Arquitectura de SAP HANA

La arquitectura de SAP HANA es fundamental para comprender cómo funciona este sistema, especialmente importante para el examen de certificación. Un entorno de SAP HANA está compuesto por muchos componentes, y los clientes no requieren todos ellos.

Concepto Clave: SAP HANA no es un producto monolítico, sino un sistema modular con una base común y diferentes añadidos según las necesidades específicas.

Ediciones de SAP HANA

SAP HANA Standard Edition

Características:

  • Versión más básica del sistema
  • Contiene el núcleo de la base de datos
  • Servicios de integración incluidos
  • Aplicaciones necesarias para trabajar con la base de datos
  • Preparada para añadir componentes adicionales mediante instalación

Es la base sobre la cual se puede construir un sistema personalizado añadiendo módulos según las necesidades del negocio.

SAP HANA Enterprise Edition

Características:

  • Versión más completa disponible
  • Incluye todo el contenido de la edición estándar
  • Muchas funcionalidades adicionales integradas
  • Mejor preparada para el desarrollo y optimización de aplicaciones sofisticadas
  • Ideal para empresas que quieren basar todo su sistema sobre HANA sin complicaciones

SAP HANA Express Edition

Características:

  • Totalmente gratuita
  • Diseñada para estudiantes y aprendizaje
  • Requisitos de hardware reducidos
  • Puede instalarse en equipos modestos
  • Disponible para descarga directa desde el portal SAP
  • Opción de despliegue en proveedores cloud certificados
Límites de Uso:
  • Hasta 32 GB de memoria RAM sin costo
  • Posible ampliación a 128 GB con licencia
  • Para más memoria se requieren licencias Standard o Enterprise

Componentes del Sistema Básico

Los siguientes componentes están presentes en todas las ediciones de SAP HANA:

Base de Datos

Componente fundamental que actúa como capa de persistencia. Incluye:

  • Tablas físicas (almacenadas en disco)
  • Tablas virtuales (Smart Data Access - SDA)
  • Almacenamiento por filas y por columnas

Motores de Procesamiento

Proporcionan capacidad de procesamiento en memoria:

  • Motor de cálculo (para vistas calculadas)
  • Motor SQL (comunicación con base de datos)
  • Motor MDX (consultas multidimensionales)

Motor XS/XSA

Servidor de aplicaciones integrado:

  • XS: Versión original (HANA 1.0)
  • XSA: Extended Application Service Advanced
  • Herramientas de desarrollo y depuración
  • Permite usar HANA como plataforma independiente
PREGUNTA DE EXAMEN: XSA se basa en el estándar abierto Cloud Foundry

Interfaces de Usuario

SAP HANA Studio

  • Cliente para Windows, Mac y Linux
  • Basado en Eclipse con plugins específicos
  • Quedando en desuso
  • Solo para desarrollo con motor XS
  • Útil para tareas de administración

Web IDE for SAP HANA

  • Interfaz web para motor XSA
  • Para desarrolladores y modeladores
  • No permite tareas de administración
  • Futuro de desarrollo en HANA
PREGUNTA DE EXAMEN: Web IDE solo para XSA, HANA Studio solo para XS

SAP HANA Cockpit

  • Interfaz web para administración
  • Basado en SAP Fiori
  • Monitoreo del sistema
  • Herramientas de administración centralizadas

Componentes del Entorno Extendido

Herramientas de Aprovisionamiento de Datos

SDI y SDQ

  • SDI (Smart Data Integration): Herramienta ETL
  • SDQ (Smart Data Quality): Limpieza de datos
  • Parte del Enterprise Information Management (EIM)
PREGUNTA DE EXAMEN: Recordar las siglas SDI y SDQ

SLT

  • SAP Landscape Transformation Replication Server
  • Módulo ETL para sistemas SAP
  • Preferiblemente en servidor dedicado
  • Trabaja con SAP Data Services

Streaming Analytics y Remote Data Sync

Utilizados para transmisión continua de datos desde múltiples canales (sensores, registros, máquinas conectadas). Muy relacionado con Internet of Things (IoT).

RDS: Componente de sincronización remota para dispositivos que no pueden estar en línea continuamente.

Motores Especiales

Motor de Texto

  • Búsquedas difusas (fuzzy search)
  • Minería de texto
  • Análisis de sentimientos
  • Procesamiento de documentos

Motor Espacial

  • Análisis basado en datos espaciales
  • Consultas de localidad y distancia
  • Mapeo de zonas
  • Cálculo de rutas

Motor de Grafos

  • Datos altamente conectados en red
  • Redes sociales
  • Cadenas de producción
  • Relaciones complejas

Motor de Series Temporales

  • Datos organizados en intervalos de tiempo
  • Identificación de patrones
  • Análisis temporal

Motor Predictivo

  • Análisis profundo de patrones
  • Resultados probables
  • Anticipación de acontecimientos
  • Requiere científicos de datos especializados

Componentes Adicionales

Data Warehousing Foundation

Proporciona funcionalidad y componentes adicionales requeridos para crear un almacén de datos nativo usando solo SAP HANA. Similar a adaptar un Business Warehouse a una plataforma HANA.

Dynamic Tiering

Proporciona una capa basada en disco para complementar el nivel de memoria existente. Descarga datos menos críticos a disco, dejando más espacio en memoria para datos críticos.

Concepto de Temperatura de Datos:

  • Datos calientes: Alta frecuencia de acceso → Memoria RAM
  • Datos fríos: Baja frecuencia de acceso → Disco duro
  • Ahorro en inversión sin sacrificar velocidad en consultas urgentes

Arquitecturas de Implementación

Stack de Tres Capas

  1. Capa de Base de Datos: SAP HANA (persistencia y procesamiento)
  2. Capa de Servidor de Aplicaciones: SAP NetWeaver (lógica empresarial)
  3. Capa Cliente: Interfaces de usuario
Push Down: Siempre que sea posible, los procesamientos deben realizarse en la capa de base de datos para aprovechar la aceleración de HANA.

Stack de Dos Capas

Para Business Intelligence:

  • Elimina la capa del servidor de aplicaciones
  • Comunicación directa entre herramientas BI y HANA
  • Protocolos: JDBC, ODBC, ODBO, BICS
  • Datos adquiridos mediante componentes EIM o herramientas externas

Con Motor XS: La base de datos actúa también como servidor de aplicaciones.

Evolución XS → XSA

Motor XS (HANA 1.0)

  • Servidor de aplicaciones ligero
  • No diseñado para aplicaciones empresariales escalables
  • Soporte para JavaScript y HTML
  • Servidor web integrado

Motor XSA (HANA 2.0)

  • Versión mucho más potente
  • Basado en Cloud Foundry (estándar abierto)
  • Aplicaciones independientes de infraestructura
  • Implementación flexible (on-premises o cloud)
  • Las aplicaciones XS no son compatibles con XSA
PREGUNTAS DE EXAMEN IMPORTANTES:
  • XSA se basa en Cloud Foundry
  • Web IDE solo para XSA, HANA Studio solo para XS
  • SDI y SDQ son las siglas de Smart Data Integration y Smart Data Quality
  • Push Down: procesamiento en la capa de base de datos

Conclusión

La arquitectura de SAP HANA es modular y flexible, permitiendo implementaciones desde básicas hasta empresariales complejas. La comprensión de estos componentes y sus interrelaciones es fundamental tanto para el trabajo práctico como para la certificación.
