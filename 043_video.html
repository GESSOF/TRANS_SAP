🎯 ¿Qué son las Funciones?

Las funciones proporcionan bloques de código reutilizables para evaluar expresiones y retornar información de la base de datos. Permiten realizar operaciones complejas de manera simple y eficiente.

💡 Analogía Simple

Una función es como una máquina especializada: le das un dato de entrada, realiza una operación específica, y te devuelve un resultado. Por ejemplo: le das "hola mundo" a UPPER() y te devuelve "HOLA MUNDO".

  • Reutilización: Escribe una vez, usa muchas veces
  • Simplicidad: Operaciones complejas en una línea
  • Legibilidad: Código más claro y mantenible

🚀 Introducción a las Funciones

🎯 Concepto Fundamental

Las funciones nos proporcionan bloques de código reutilizables para evaluar expresiones y retornar información de la base de datos. Están permitidas en cualquier lugar donde se permitan expresiones SQL.

📝 Analogía del Flujo de Función

Imagina que estás programando y tienes código que usas frecuentemente:

-- En lugar de escribir esto muchas veces: valor1 + valor2 * 100 / valor3 - valor4 -- Creas una función F que lo encapsula: F(valor1, valor2, valor3, valor4) -- Flujo: Código → Llama F → Ejecuta operación → Devuelve resultado → Continúa

🔧 Características de las Funciones

  • Misma sintaxis que SQL: Mayúsculas, minúsculas permitidas
  • Deben empezar por letra o guion bajo
  • Caracteres especiales entre comillas dobles
  • Pueden contener comandos HANA SQL internamente
  • Reutilizables infinitas veces

💡 Diferencia Importante

Funciones estándar: Las que veremos ahora, ya incluidas en HANA SQL.

Funciones personalizadas: Las crearemos más adelante con SQL Script (lenguaje de programación avanzado).

📝 Sintaxis General de Funciones

🔧 Estructura Básica

Todas las funciones siguen el mismo patrón sintáctico básico:

NOMBRE_FUNCION(parámetro1, parámetro2, ...)

📋 Componentes

  • NOMBRE_FUNCION: Identificador de la función (obligatorio)
  • Paréntesis: Requeridos, aunque estén vacíos
  • Parámetros: Opcionales, depende de cada función
  • Separador: Coma entre múltiples parámetros

✅ Ejemplos de Sintaxis

-- Función sin parámetros SELECT CURRENT_DATE FROM DUMMY; -- Función con un parámetro SELECT ABS(-25) FROM DUMMY; -- Función con múltiples parámetros SELECT CONCAT('Hola', ' Mundo') FROM DUMMY; -- Función anidada (función dentro de función) SELECT LENGTH(TRIM(' Hola ')) FROM DUMMY;

⚠️ Reglas Importantes

  • Paréntesis obligatorios: Aunque la función no tenga parámetros
  • Excepción: Algunas funciones de sistema (CURRENT_DATE) permiten omitirlos
  • Parámetros específicos: Cada función define qué acepta
  • Orden importante: Los parámetros deben estar en el orden correcto
  • Tipos compatibles: Los parámetros deben ser del tipo esperado

🚨 Funciones de Sistema Especiales

Algunas funciones como CURRENT_DATE o CURRENT_TIME pueden generar error si les agregas paréntesis vacíos. Son excepciones al comportamiento estándar.

💡 Flexibilidad de Uso

Las funciones pueden usarse en cualquier lugar donde se permitan expresiones: SELECT, WHERE, HAVING, ORDER BY, GROUP BY, etc.

🔄 Funciones de Conversión de Tipos

🎯 Propósito

Estas funciones nos permiten cambiar el tipo de dato de un valor para poder realizar operaciones específicas que requieren un tipo particular.

CAST

Conversión universal a cualquier tipo

CAST(expresión AS tipo_dato)

TO_BIGINT

Convierte a entero grande

TO_BIGINT('123')

TO_CHAR

Convierte a cadena de caracteres

TO_CHAR(123)

📝 Caso Práctico: Texto a Número

Tienes números almacenados como texto y necesitas hacer operaciones matemáticas:

-- ❌ Esto fallaría si fueran letras SELECT '52' + '46' FROM DUMMY; -- Resultado: 98 (HANA convierte automáticamente números como texto) -- ❌ Esto da error porque no son números válidos SELECT 'Hola' + 'Mundo' FROM DUMMY; -- ERROR: No se puede convertir texto a número -- ✅ Conversión explícita segura SELECT TO_BIGINT('52') + TO_BIGINT('46') FROM DUMMY; -- Resultado: 98

🔧 CAST - La Función Universal

CAST puede convertir a cualquier tipo de dato. Su sintaxis es:

-- Sintaxis de CAST CAST(expresión AS tipo_dato) -- Ejemplos prácticos SELECT CAST('123' AS INTEGER) FROM DUMMY; SELECT CAST(123 AS VARCHAR(10)) FROM DUMMY; -- ❌ Conversión que puede fallar SELECT CAST('25' AS ALPHANUM) FROM DUMMY; -- ALPHANUM son números pero se comportan como texto

⚠️ Cuidado con ALPHANUM

ALPHANUM son números almacenados como cadena de caracteres. No permiten operaciones matemáticas directas, aunque contengan números.

💡 Regla de Conversión

Texto → Número: Solo funciona si el texto contiene números válidos.

Número → Texto: Siempre funciona, cualquier número se puede representar como texto.

📅 Funciones de Fecha y Hora

🎯 ¿Por Qué Son Necesarias?

Con datos de tipo fecha no puedes hacer operaciones directas como sumar o restar. Necesitas funciones especializadas para cualquier cálculo con fechas.

CURRENT_DATE

Fecha actual del sistema

CURRENT_DATE

CURRENT_TIME

Hora actual del sistema

CURRENT_TIME

ADD_DAYS

Añade días a una fecha

ADD_DAYS(fecha, número_días)

ADD_MONTHS

Añade meses a una fecha

ADD_MONTHS(fecha, número_meses)

ADD_YEARS

Añade años a una fecha

ADD_YEARS(fecha, número_años)

DAYS_BETWEEN

Días entre dos fechas

DAYS_BETWEEN(fecha1, fecha2)

📝 Ejemplos Prácticos

-- ❌ Esto NO funciona con fechas SELECT CURRENT_DATE + 2 FROM DUMMY; -- ERROR: No se pueden sumar números a fechas directamente -- ✅ Forma correcta de añadir días SELECT ADD_DAYS(CURRENT_DATE, 2) FROM DUMMY; -- Si hoy es 29/12/2024, resultado: 31/12/2024 -- ✅ Añadir meses (con ajuste automático) SELECT ADD_MONTHS(CURRENT_DATE, 2) FROM DUMMY; -- 29/12/2024 → 28/02/2025 (febrero tiene menos días) -- ✅ Calcular diferencia entre fechas SELECT DAYS_BETWEEN( ADD_MONTHS(CURRENT_DATE, 2), CURRENT_DATE ) FROM DUMMY; -- Resultado: 61 días (aproximadamente)

🔧 Funciones Anidadas

Puedes combinar funciones de fecha para cálculos complejos:

-- Cálculo complejo: días entre hoy y dentro de 2 meses SELECT DAYS_BETWEEN( ADD_MONTHS(CURRENT_DATE, 2), -- Fecha futura CURRENT_DATE -- Fecha actual ) FROM DUMMY; -- Si el resultado es negativo, invierte el orden SELECT DAYS_BETWEEN( CURRENT_DATE, -- Fecha menor primero ADD_MONTHS(CURRENT_DATE, 2) -- Fecha mayor segundo ) FROM DUMMY;

⚠️ Cuidado con Paréntesis

Algunas funciones de fecha como CURRENT_DATE pueden dar error si agregas paréntesis vacíos. Otras como ADD_DAYS los requieren obligatoriamente.

💡 Ajuste Automático

Las funciones de fecha son inteligentes. Si añades meses y el día resultante no existe (ej: 31 de febrero), automáticamente ajusta al último día válido del mes.

📝 Funciones de Cadena

🎯 Manipulación de Texto

Estas funciones nos permiten procesar y manipular cadenas de caracteres de múltiples formas: concatenar, extraer partes, limpiar espacios, medir longitud, etc.

CONCAT

Une dos cadenas de texto

CONCAT('Hola', ' Mundo')

LEFT

Extrae caracteres desde el inicio

LEFT('Hola Mundo', 4)

LENGTH

Cuenta caracteres en la cadena

LENGTH('Hola Mundo')

TRIM

Elimina espacios al inicio y final

TRIM(' Hola ')

📝 CONCAT vs Operador ||

Existen dos formas de concatenar texto:

-- Método 1: Operador || (más simple) SELECT 'Hola ' || '¿Qué tal?' FROM DUMMY; -- Resultado: 'Hola ¿Qué tal?' -- Método 2: Función CONCAT SELECT CONCAT('Hola ', '¿Qué tal?') FROM DUMMY; -- Resultado: 'Hola ¿Qué tal?' -- ❌ Limitación de CONCAT: solo une DOS cadenas -- Para tres o más, necesitas anidar: SELECT CONCAT( CONCAT('Hola ', '¿Qué '), 'tal?' ) FROM DUMMY; -- ✅ Con || es más simple para múltiples cadenas SELECT 'Hola ' || '¿Qué ' || 'tal?' FROM DUMMY;

🔧 LEFT - Extracción de Caracteres

LEFT extrae un número específico de caracteres desde el inicio de la cadena:

-- Extraer los primeros 4 caracteres SELECT LEFT('Hola ¿Qué tal?', 4) FROM DUMMY; -- Resultado: 'Hola' -- Útil para códigos, prefijos, etc. SELECT LEFT('ESP-001-2024', 3) FROM DUMMY; -- Resultado: 'ESP'

📏 LENGTH - Contar Caracteres

LENGTH devuelve el número total de caracteres en una cadena:

-- Contar caracteres incluyendo espacios SELECT LENGTH('Hola ¿Qué tal?') FROM DUMMY; -- Resultado: 15 (H-o-l-a-espacio-¿-Q-u-é-espacio-t-a-l-?) -- Útil para validaciones SELECT * FROM usuarios WHERE LENGTH(password) < 8; -- Encuentra contraseñas muy cortas

✨ TRIM - Limpieza de Espacios

TRIM elimina espacios innecesarios al inicio y final de las cadenas:

-- Demostrar el problema de espacios SELECT LENGTH(' Hola ¿Qué tal? ') FROM DUMMY; -- Resultado: 21 caracteres (incluye espacios extras) -- Limpiar espacios con TRIM SELECT TRIM(' Hola ¿Qué tal? ') FROM DUMMY; -- Resultado: 'Hola ¿Qué tal?' (sin espacios extremos) -- Verificar la diferencia en longitud SELECT LENGTH(TRIM(' Hola ¿Qué tal? ')) FROM DUMMY; -- Resultado: 15 caracteres (espacios eliminados)

🔍 Funciones Anidadas

Puedes combinar funciones de cadena para operaciones complejas. Ten cuidado con el balance de paréntesis cuando anides múltiples funciones.

💡 Aplicación Práctica

Estas funciones son especialmente útiles para limpiar datos, validar formatos, extraer códigos, y preparar información para presentación o análisis.

🔢 Funciones Numéricas

🎯 Operaciones Matemáticas Avanzadas

Estas funciones nos permiten realizar operaciones matemáticas complejas más allá de las básicas (+, -, *, /). Incluyen funciones trigonométricas, valor absoluto, redondeo, etc.

ABS

Valor absoluto (sin signo)

ABS(-25) = 25

COS

Coseno de un ángulo

COS(0) = 1

SIN

Seno de un ángulo

SIN(1.5708) ≈ 1

TAN

Tangente de un ángulo

TAN(0.785) ≈ 1

📝 ABS - Valor Absoluto

La función más práctica para uso general. Elimina el signo negativo:

-- Valor absoluto básico SELECT ABS(-25) FROM DUMMY; -- Resultado: 25 -- Con números positivos no cambia nada SELECT ABS(25) FROM DUMMY; -- Resultado: 25 -- Útil para calcular diferencias SELECT ABS(ventas_mes1 - ventas_mes2) AS diferencia FROM estadisticas; -- Siempre devuelve diferencia positiva

📐 Funciones Trigonométricas

Son más especializadas y se usan principalmente en cálculos científicos, ingeniería, o análisis geométricos:

-- Coseno (para ángulos en radianes) SELECT COS(0) FROM DUMMY; -- Resultado: 1 -- Seno SELECT SIN(1.5708) FROM DUMMY; -- Resultado: ≈ 1 (90 grados en radianes) -- Tangente SELECT TAN(0.785) FROM DUMMY; -- Resultado: ≈ 1 (45 grados en radianes)

⚠️ Funciones Especializadas

Las funciones trigonométricas son muy específicas. En aplicaciones de negocio típicas, ABS es mucho más útil que COS, SIN o TAN.

🎯 Casos de Uso Prácticos

  • ABS: Diferencias, distancias, variaciones
  • Trigonométricas: Cálculos geográficos, análisis científicos
  • Combinadas: Fórmulas matemáticas complejas

💡 Recomendación

Para aplicaciones de negocio típicas, enfócate en ABS. Las funciones trigonométricas las usarás solo en casos muy específicos.

🔧 Funciones de Sistema

🎯 Información del Entorno

Estas funciones nos proporcionan información sobre la sesión actual, el usuario conectado, el esquema en uso, y otros datos del sistema que pueden ser útiles para lógica condicional o auditoría.

CURRENT_USER

Usuario técnico de la sesión

CURRENT_USER

CURRENT_SCHEMA

Esquema activo por defecto

CURRENT_SCHEMA

IFNULL

Control de valores nulos

IFNULL(campo, 'N/A')

📝 CURRENT_USER - Usuario de Sesión

Muestra el usuario técnico que está ejecutando las consultas:

-- Ver usuario técnico actual SELECT CURRENT_USER FROM DUMMY; -- Resultado: Usuario técnico (puede ser diferente al usuario de login) -- Útil para lógica condicional SELECT CASE WHEN CURRENT_USER = 'ADMIN_USER' THEN 'Acceso total' ELSE 'Acceso limitado' END AS nivel_acceso FROM DUMMY;

🗂️ CURRENT_SCHEMA - Esquema Activo

Muestra el esquema al que estamos conectados por defecto en esta sesión:

-- Ver esquema actual SELECT CURRENT_SCHEMA FROM DUMMY; -- Resultado: Nombre del esquema por defecto -- Útil para queries dinámicas o validaciones SELECT 'Trabajando en esquema: ' || CURRENT_SCHEMA AS info FROM DUMMY;

⚠️ Usuario Técnico vs Usuario Real

CURRENT_USER muestra el usuario técnico que interactúa con la base de datos, que puede ser diferente al usuario con el que te logueaste en la aplicación.

🔧 IFNULL - Control de Nulos

Aunque se verá más adelante en detalle, IFNULL maneja valores nulos:

-- Reemplazar nulos con un valor por defecto SELECT IFNULL(campo_opcional, 'Sin datos') FROM tabla; -- Más ejemplos en temas de control de flujo
🎯 Casos de Uso
  • Auditoría: Registrar qué usuario ejecutó qué acción
  • Seguridad: Lógica condicional basada en usuario
  • Configuración: Comportamiento específico por esquema
  • Debugging: Verificar contexto de ejecución

💡 Información Contextual

Estas funciones son valiosas para crear aplicaciones que se adapten al contexto de ejecución, proporcionando diferentes comportamientos según el usuario o entorno.
