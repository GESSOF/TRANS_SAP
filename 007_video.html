🔄 Alta Disponibilidad (High Availability)

💡 Concepto

La alta disponibilidad busca que múltiples usuarios puedan acceder al sistema simultáneamente sin bloqueos, incluyendo tolerancia a desastres.

Analogía: Como un generador de emergencia en un hospital - si falla el sistema principal, inmediatamente se activa el respaldo.

En multinacionales con servidores en diferentes países (España, EE.UU., Inglaterra), si falla un data center completo, el tráfico se desvía automáticamente a otro.

👤 Autenticación

🔍 Definición

Proceso de verificación de identidad cuando un usuario intenta conectarse a la base de datos HANA. Comprueba si realmente tiene permisos para acceder.

🔐 Métodos de Autenticación

🔑 Usuario/Contraseña

Método tradicional más común

🎫 Kerberos

Protocolo avanzado del MIT

🎫 Protocolo Kerberos

Creado por el MIT - Permite a dos ordenadores en una red insegura demostrar su identidad mutuamente de manera segura.

🏗️ Componentes Clave

  • KDC (Key Distribution Center): Centro de distribución de claves
  • AS (Authentication Server): Servidor de autenticación
  • TGS (Ticket Granting Server): Servidor emisor de tickets
  • Criptografía simétrica: Base del protocolo

1️⃣ Autenticación inicial

Cliente se autentica contra Authentication Server

2️⃣ Solicitud de ticket

Demuestra al TGS que está autorizado para recibir ticket

3️⃣ Acceso al servicio

Demuestra al servidor que ha sido aprobado para usar el servicio

📦 Sistema de Roles

Un rol es como una "caja" que contiene un conjunto de privilegios predefinidos para un perfil específico de trabajo.

👨‍💻 Desarrollador

• Acceso a tablas específicas
• Permisos de ejecución
• Creación de objetos

🎨 Modelador

• Permisos de creación en esquemas
• Consulta de datos
• Diseño de modelos

👤 Usuario Normal

• Consulta básica
• Lectura de reportes
• Sin permisos de modificación

⚙️ Proceso de Asignación

  • Paso 1: Crear rol base para el perfil
  • Paso 2: Asignar rol al usuario nuevo
  • Paso 3: Añadir privilegios específicos individuales
  • Ventaja: Eficiencia y consistencia en permisos

🔐 Cifrado y Encriptación

🛡️ Objetivo del Cifrado

Garantizar que la información permanezca segura y solo sea legible por usuarios con permisos y autenticación correcta.

📈 Evolución del Cifrado

⚙️ Máquina Enigma

Segunda Guerra Mundial - Cambio de posición de letras

💻 Cifrado Moderno

Algoritmos complejos - Extremadamente difíciles de descifrar

🔒 Cifrado Actual

Protección contra ataques de hackers y secuestro de datos

🔍 Cifrado en la Práctica

Cuando los hackers roban datos de grandes empresas, frecuentemente no pueden utilizarlos porque están encriptados. El tiempo necesario para descifrarlos sin la clave puede ser de décadas o siglos.

💡 Principio Básico

  • Sin credenciales válidas = Información ilegible
  • Con autenticación correcta = Información legible
  • Protección contra acceso no autorizado
  • Última línea de defensa en seguridad

📊 Audit Logging (Registros de Auditoría)

📝 Definición

También conocidos como "logs" o "trazas del sistema" - Registran un seguimiento completo de todas las actividades en el sistema.

🔍 Qué se Registra

🔐 Accesos

Recursos, datos, funciones, procedimientos

👥 Administración

Usuarios, roles, parámetros de seguridad

📋 Reglas

Configuraciones y políticas del sistema

🎯 Beneficios de la Auditoría

✅ Ventajas

  • Trazabilidad completa: Saber quién hizo qué y cuándo
  • Detección de anomalías: Identificar comportamientos sospechosos
  • Cumplimiento normativo: Satisfacer requisitos legales
  • Investigación de incidentes: Análisis forense de problemas
  • Optimización: Identificar patrones de uso del sistema
