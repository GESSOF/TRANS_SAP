🏗️ Evolución Arquitectónica: Tradicional vs HANA

🐌 Sistema Tradicional

  • Almacenamiento: Disco duro principal
  • Memoria RAM: Cache limitada
  • Cuello de botella: Disco duro → RAM
  • Procesamiento: Secuencial
  • Duplicación: Sistemas separados (OLTP/OLAP)

⚡ Sistema SAP HANA

  • Almacenamiento: Memoria RAM principal
  • Disco duro: Solo respaldo
  • Cuello de botella: RAM → CPU
  • Procesamiento: Paralelo (multicore)
  • Unificación: Base de datos única

🔐 Principios ACID de las Bases de Datos

Todo sistema de base de datos debe cumplir los principios ACID para garantizar la integridad de los datos:

A.C.I.D

Atomicity • Consistency • Isolation • Durability

🔸 Atomicidad

Concepto: La transacción se ejecuta completamente o no se ejecuta nada.

Ejemplo: Cajero automático - si hay error, se revierten todas las operaciones (devolver dinero a la cuenta si no se entregó físicamente).

🔸 Consistencia

Concepto: Los datos mantienen su validez durante las operaciones.

Mecanismo: Bloqueos que impiden modificaciones simultáneas del mismo registro.

Ejemplo: Dos transacciones no pueden modificar el mismo saldo bancario simultáneamente.

🔸 Aislamiento

Concepto: Las transacciones son independientes entre sí.

Funcionamiento: Los cambios no son visibles hasta confirmar la transacción completa.

Beneficio: Evita lecturas de datos en estados transitorios.

🔸 Durabilidad

Concepto: Los datos confirmados persisten ante cualquier fallo.

Garantía: Si se guarda información, debe estar disponible después de un reinicio.

Implementación: Respaldos automáticos y recuperación de fallos.

🌡️ Temperatura de los Datos en SAP HANA

📊 Clasificación Térmica de la Información

SAP HANA clasifica los datos según su frecuencia de uso para optimizar el rendimiento:

🔥 Datos Calientes

  • Ubicación: Memoria RAM
  • Características: Uso continuo
  • Ejemplo: Datos del año actual
  • Objetivo: Acceso óptimo a CPU

🌡️ Datos Templados

  • Ubicación: Disco duro
  • Características: Uso ocasional
  • Ejemplo: Datos de últimos 5 años
  • Comportamiento: Se mueven a RAM cuando se solicitan

❄️ Datos Fríos

  • Ubicación: Almacenamiento archivado
  • Características: Uso muy raro
  • Ejemplo: Datos históricos +5 años
  • Propósito: Conservación legal/regulatoria

📥 Concepto Push Down: Revolución en el Desarrollo

🔄 Cambio de Filosofía de Desarrollo

Push Down: "Empujar hacia abajo" - mover el procesamiento a la base de datos

📤 Desarrollo Tradicional

  1. Extraer datos brutos de la BD
  2. Procesar en el servidor de aplicaciones
  3. Realizar cálculos complejos
  4. Ordenar y agregar datos
  5. Devolver resultados o guardar

Objetivo: Liberar recursos de la BD

📥 Desarrollo con HANA (Push Down)

  1. Realizar cálculos en la BD
  2. Procesar agregaciones en HANA
  3. Aplicar ordenamientos complejos
  4. Recibir resultados finales
  5. Solo presentar datos al usuario

Beneficio: Aprovechar la potencia de HANA

💡 Ventajas del Push Down

  • Código simplificado: Menos lógica en aplicaciones
  • Recursos liberados: Servidor de aplicaciones solo presenta datos
  • Transferencia mínima: Solo se envían resultados finales
  • Performance optimizado: Aprovecha la velocidad de HANA

🔄 OLTP, OLAP y la Revolución HTAP

📋 Sistemas Tradicionales Separados:

💼 OLTP (Online Transaction Processing)

  • Función: Sistemas transaccionales
  • Optimización: Lectura/escritura rápida
  • Nivel: Registro individual
  • Uso: Operaciones diarias del negocio

📊 OLAP (Online Analytical Processing)

  • Función: Sistemas analíticos
  • Optimización: Carga por lotes y agregaciones
  • Nivel: Lectura intensiva masiva
  • Uso: Business Intelligence y reportes

🎯 Problema Tradicional

Necesidad de duplicar datos entre sistema OLTP (producción) y OLAP (análisis), causando:

  • Redundancia de información
  • Procesos ETL complejos
  • Retrasos en la actualización
  • Mayores costos de mantenimiento
HTAP

Hybrid Transaction Analytical Processing

La respuesta de SAP HANA: Una sola plataforma para ambos propósitos

✅ Beneficios del HTAP

  • Eliminación de redundancias de datos
  • Una sola copia de datos para cualquier acceso
  • Procesamiento transaccional y analítico simultáneo
  • Datos siempre actualizados en tiempo real
  • Reducción significativa de la complejidad del sistema

🔑 Conceptos Clave para Recordar

  • In-Memory: Tecnología que mantiene toda la BD en memoria RAM
  • Push Down: Mover procesamiento desde aplicación hacia base de datos
  • ACID: Principios fundamentales: Atomicidad, Consistencia, Aislamiento, Durabilidad
  • HTAP: Hybrid Transaction Analytical Processing - unión de OLTP y OLAP
  • Cuello de Botella: Cambio de disco→RAM a RAM→CPU
  • Temperatura de Datos: Calientes (RAM), Templados (Disco), Fríos (Archivo)
